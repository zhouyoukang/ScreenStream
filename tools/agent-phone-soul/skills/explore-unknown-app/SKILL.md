# 探索陌生APP

## 哲学基础

> 你不需要预先知道每个APP怎么用。
> 你需要知道如何面对任何陌生的界面，通过观察理解它，通过尝试学会它。
> 人类用户也是这样学会新APP的：看 → 理解 → 试 → 看结果。

## 触发条件
- 需要在一个 Memory 中没有操作记录的 APP 中完成任务
- 打开一个 APP 后不确定如何导航到目标功能

## 探索策略

### Phase 1: 第一印象（5秒）

打开APP后立即做三件事：
```
1. GET /foreground          → 确认包名
2. GET /screen/text         → 获取首屏所有文字和可点击元素
3. GET /viewtree?depth=4    → 获取浅层结构
```

从这三个信息中提取：
- **APP身份**：包名、首屏标题
- **主要导航结构**：底部Tab栏？侧边抽屉？顶部Tab？
- **核心功能入口**：可点击的按钮/链接/菜单项
- **当前页面类型**：首页/登录页/引导页/权限请求页

### Phase 2: 处理障碍（如有）

如果首屏不是正常的APP主页：
- **登录页** → 停止，告知用户"APP需要登录，请手动登录后告诉我"
- **引导页** → 尝试 find-click("跳过"/"skip"/"下一步"/"知道了")
- **权限弹窗** → find-click("允许"/"allow") 或 dismiss
- **更新提示** → find-click("以后再说"/"稍后"/"取消") 或 dismiss

### Phase 3: 建立心智模型（10秒）

理解APP的导航模式：

**模式A: 底部Tab导航**（微信、淘宝、抖音风格）
- 特征：屏幕底部有 3-5 个可点击文字/图标
- 策略：逐个点击底部Tab，记录每个Tab的页面内容

**模式B: 列表+详情**（设置、文件管理器风格）
- 特征：屏幕上是可滚动的列表项
- 策略：滚动浏览列表，找到与目标相关的项

**模式C: 搜索入口**（应用商店、地图风格）
- 特征：顶部有搜索框/搜索图标
- 策略：优先使用搜索功能直达目标

**模式D: 功能网格**（支付宝首页风格）
- 特征：九宫格/多行多列的功能入口
- 策略：扫描所有可见入口文字，找到最匹配的

### Phase 4: 导航到目标

基于 Phase 3 建立的心智模型：

1. 如果目标功能在当前页面可见 → 直接 find-click
2. 如果有搜索功能 → 搜索目标关键词
3. 如果是Tab导航 → 猜测目标可能在哪个Tab下
4. 如果是列表 → 滚动查找
5. 以上都不行 → 尝试 POST /command {"command": "找到<目标功能>"}

### Phase 5: 记录探索结果

无论成功或失败，记录到 Memory：

```
## [APP名] 首次探索记录
- 包名：
- 导航模式：[Tab/列表/搜索/网格]
- 首屏关键元素：[列表]
- 已发现的功能入口：[列表]
- 成功路径：[从首页到目标功能的点击序列]
- 失败尝试：[尝试过但失败的路径]
- 设备：
- 日期：
```

## 探索伦理

- **不要盲目点击**：每次点击前要有明确的理由（"这个按钮的文字与目标相关"）
- **不要深入超过3层**：如果3次导航后仍未找到目标，回退到首页重新规划
- **随时准备回退**：每次进入新页面前，确认知道如何返回（通常是 POST /back）
- **遇到敏感页面立即后退**：看到"支付"、"转账"、"授权"等关键词 → POST /back

## 从探索到熟练

```
第1次：完整执行 Phase 1-5（探索模式，15-30秒）
第2次：跳过 Phase 1-3，直接用记录的路径（半熟练，5-10秒）
第3次+：直接执行记录的路径（熟练，2-5秒）
```

这就是渐进抽象在操作层面的体现：
具体操作 → 操作路径 → 导航模式 → APP心智模型
