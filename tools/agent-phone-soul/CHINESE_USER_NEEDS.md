## 成本效率分析

```
Level 0 覆盖: ~70% 日常需求（零成本）
Level 1 覆盖: ~20% 日常需求（正则匹配，极低成本）
Level 2 覆盖: ~8% 日常需求（小模型，低成本）
Level 3 覆盖: ~2% 日常需求（大模型，高成本）

→ 90%需求可以零/极低成本解决
→ 只有10%需要AI模型介入
```

## 核心洞察

1. **中国用户最常用的是“查询+开关”类指令**——这些全部可以Level 0解决
2. **APP启动是第二大类需求**——只需要包名映射表
3. **通讯类（打电话/发消息）是第三大类**——Intent可以直接实现
4. **真正需要AI的是“理解+决策+创造”类**——占比极小但价值最高
5. **中国用户特色**：微信是绝对核心（消息/支付/扫码/小程序），其次淘宝/支付宝/高德
6. **剪贴板PC↔手机同步**是竞品核心卖点——已实现POST/GET /clipboard
