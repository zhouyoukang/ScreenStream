## 🔧 配置步骤

### 步骤 1: 启动所有语音服务

运行脚本: `🔧_工具脚本\启动语音唤醒服务.bat`

或手动执行:
```powershell
& 'C:\Program Files\Docker\Docker\resources\bin\docker.exe' start romantic_colden kind_knuth custom_wake wake_zh vibrant_hamilton
```

### 步骤 2: 检查 Wyoming 集成配置

当前已配置的 Wyoming 集成：
- ✅ **faster-whisper**: 192.168.31.141:10310 (主 STT)
- ✅ **openwakeword**: 192.168.31.141:10400
- ✅ **openwakeword**: 192.168.31.141:10443
- ✅ **piper**: 192.168.31.141:10200

如需添加新服务：
1. 打开 **设置** → **设备与服务** → **添加集成**
2. 搜索 **Wyoming**
3. 输入主机和端口

### 步骤 3: 配置语音助手管道

1. 打开 **设置** → **语音助手**
2. 编辑或创建管道，设置：
   - **语音识别 (STT)**: `faster-whisper` 或 `Home Assistant Cloud`
   - **唤醒词**: `openwakeword` → 选择唤醒词 (如 `hey_jarvis`, `alexa`, `ok_nabu`)
   - **对话引擎**: 选择你的 AI (如 qwen3, ChatGPT 等)
   - **语音合成 (TTS)**: `edge_tts` 或 `piper`

## 🐛 常见问题排查

### 问题 1: 唤醒词不响应

**检查:**
1. OpenWakeWord 容器是否运行: `docker ps | grep openwakeword`
2. HA 中是否配置了 Wyoming 集成
3. 管道中是否选择了唤醒词

**解决:**
```powershell
# 重启唤醒词服务
docker restart custom_wake wake_zh
```

### 问题 2: 语音识别不工作

**检查:**
1. Whisper 容器是否运行: `docker ps | grep whisper`
2. HA 中是否有 `stt.faster_whisper` 实体

**解决:**
```powershell
# 重启 Whisper
docker restart romantic_colden

# 查看日志
docker logs romantic_colden --tail 20
```

### 问题 3: Wyoming 集成显示不可用

**原因:** 服务端口不可达

**解决:**
1. 确保防火墙允许端口 10200, 10300, 10400, 10443
2. 检查容器 IP 绑定是否正确
3. 重启容器并等待 30 秒
