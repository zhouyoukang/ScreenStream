## 🏗️ **推荐的集成架构**

### 📐 **三层架构设计**

```
┌─────────────────────────────────────────────────────────────┐
│                      n8n 智能编排层                         │
│  🧠 AI决策  📊 数据分析  🔗 外部集成  📱 统一API           │
└─────────────────────┬───────────────────────────────────────┘
                      │ REST API / Webhook
┌─────────────────────┴───────────────────────────────────────┐
│                   现有自动化层                               │
│  Node-RED (设备控制) ←→ Home Assistant (状态管理)          │
└─────────────────────┬───────────────────────────────────────┘
                      │ 设备协议 (Zigbee/WiFi/HTTP)
┌─────────────────────┴───────────────────────────────────────┐
│                     物理设备层                               │
│  Sonoff • 小米 • 飞利浦 • AI语音 • 音乐系统                │
└─────────────────────────────────────────────────────────────┘
```

### 🔌 **接入方式**

#### 1. **保持现有系统不变**
- Node-RED 继续处理设备控制
- Home Assistant 继续管理设备状态
- n8n 作为上层编排和增强

#### 2. **通过API桥接**
- n8n → Node-RED: `http://localhost:1880/scene-control`
- n8n → Home Assistant: `http://localhost:8123/api/services/`
- n8n → 外部控制: `http://192.168.31.228:8080/`

#### 3. **数据双向同步**
- 状态变化通知: HA → Node-RED → n8n
- 场景触发: n8n → Node-RED → HA
- 外部控制: n8n → 外部系统 → 设备
