## 二、可编程接口清单

### 2.1 本地RPC接口（核心 — 无需企业版）

向日葵Service运行时，在随机高端口上暴露HTTP接口：

```
GET  http://127.0.0.1:{PORT}/cgi-bin/rpc?action=verify-haras
     → 返回 CID (会话凭证)

POST http://127.0.0.1:{PORT}/cgi-bin/rpc?action=verify-haras
     Cookie: CID={session}
     → 验证会话

GET  http://127.0.0.1:{PORT}/cgi-bin/rpc?action=get-status
     → 获取状态信息（识别码/在线状态）
```

**发现端口的方法：**
```powershell
# 扫描向日葵的随机端口
$proc = Get-Process SunloginClient -ErrorAction SilentlyContinue | Select-Object -First 1
if ($proc) {
    Get-NetTCPConnection -OwningProcess $proc.Id -State Listen -ErrorAction SilentlyContinue |
    Where-Object { $_.LocalPort -gt 40000 } |
    Select-Object LocalPort
}
```

## 三、Agent五感控制方案

### 3.1 方案对比

| 方案 | 成本 | 五感覆盖 | 不干扰本机 | 推荐 |
|------|------|---------|-----------|------|
| **A. 本地RPC+remote_agent** | 零 | ⭐⭐⭐⭐⭐ | ✅ | ⭐⭐⭐⭐⭐ |
| B. 企业版OpenAPI | 付费 | ⭐⭐⭐⭐ | ✅ | ⭐⭐⭐ |
| C. 向日葵+remote_agent混合 | 零 | ⭐⭐⭐⭐⭐ | ✅ | ⭐⭐⭐⭐ |
| D. SLAPI SDK嵌入 | 开发成本高 | ⭐⭐⭐⭐⭐ | ✅ | ⭐⭐ |

### 3.2 推荐方案：A+C 混合（零成本最大能力）

**核心思路**：向日葵负责**穿透网络**（P2P/中继），remote_agent负责**五感控制**。

```
┌─────────────────────────────────────────────┐
│  AI Agent (本机 Cascade)                     │
│  ├── 直连模式: remote_agent HTTP API         │
│  └── 穿透模式: 向日葵端口转发 → remote_agent │
└──────┬──────────────────────────┬────────────┘
       │ 局域网直连               │ 公网穿透
       ▼                         ▼
┌──────────────┐  ┌─────────────────────────┐
│ remote_agent │  │ 向日葵端口转发            │
│ :9903        │  │ 公网 → 127.0.0.1:9903   │
│ 五感全覆盖   │  │ (config.ini [forward])   │
└──────────────┘  └─────────────────────────┘
```

### 3.3 五感映射

| 感官 | 向日葵提供 | remote_agent提供 | Agent调用 |
|------|-----------|-----------------|----------|
| 👁 视觉 | 屏幕投射(视频流) | /screenshot + /windows | HTTP GET |
| 👂 听觉 | 音频转发 | (系统音频API) | — |
| 🖐 触觉 | 键鼠透传 | /key /click /type /scroll /drag | HTTP POST |
| 🧠 认知 | CMD远控 | /shell /processes /sysinfo /files | HTTP GET/POST |
| 🎮 管理 | 远程开机/黑屏 | /power /service /network | HTTP POST |

### 3.4 向日葵独有能力（remote_agent无法替代）

1. **公网穿透** — P2P+中继，无需公网IP/Cloudflare Tunnel
2. **远程开机** — 搭配向日葵插座，WoL唤醒
3. **黑屏模式** — oraydpms.sys驱动级黑屏，被控端不可见
4. **虚拟显示器** — OrayIddDriver，无头机器也有桌面
5. **文件传输** — 内置高速文件通道
6. **端口转发** — config.ini已配置RDP:3389和路由器:80

## 四、实操步骤（Agent自动化远控）

### 4.1 局域网模式（当前环境）

当前PC和远程PC在同一局域网，直接用remote_agent:
```python
# remote_agent已部署在远程PC(127.0.0.2:9903)
import requests
r = requests.get("http://127.0.0.2:9903/screenshot")
r = requests.post("http://127.0.0.2:9903/key", json={"key": "enter"})
r = requests.post("http://127.0.0.2:9903/shell", json={"command": "dir"})
```

### 4.2 公网模式（借助向日葵穿透）

```ini
# 在远程PC的 config.ini [forward] 中添加:
channels=name=RemoteAgent&pro=&addr=127.0.0.1:9903;
```

然后通过向日葵的端口转发功能，从任意公网位置访问remote_agent。

### 4.3 向日葵本地RPC调用（高级）

```python
import socket, requests

def find_sunlogin_port():
    """扫描向日葵的随机高端口"""
    for port in range(40000, 65536):
        try:
            s = socket.socket()
            s.settimeout(0.05)
            s.connect(('127.0.0.1', port))
            s.close()
            # 验证是否为向日葵
            try:
                r = requests.get(f'http://127.0.0.1:{port}/cgi-bin/rpc?action=verify-haras', timeout=1)
                if r.status_code == 200:
                    return port
            except:
                pass
        except:
            pass
    return None

port = find_sunlogin_port()
if port:
    # 获取会话
    r = requests.get(f'http://127.0.0.1:{port}/cgi-bin/rpc?action=verify-haras')
    cid = r.cookies.get('CID')
    print(f"向日葵端口: {port}, CID: {cid}")
```

## 六、与现有remote_agent的整合建议

### 最优组合

| 场景 | 方案 | 延迟 |
|------|------|------|
| **局域网** | remote_agent直连 | <5ms |
| **同机跨账号** | PSRemoting + remote_agent | <10ms |
| **公网(有Cloudflare)** | Cloudflare Tunnel → remote_agent | ~50ms |
| **公网(无Tunnel)** | 向日葵端口转发 → remote_agent | ~100ms |
| **断网恢复** | 向日葵远程开机 → 启动remote_agent | 30s+ |

### 向日葵的不可替代价值

1. **P2P穿透** — 复杂NAT环境下仍可连接（Oray中继服务器保底）
2. **远程开机** — 搭配硬件插座，电源级别唤醒
3. **视频流** — 高帧率(最高144fps)屏幕投射（企业版）
4. **黑屏被控** — 驱动级隐藏屏幕，旁人看不到操作
5. **虚拟显示器** — 服务器/无头机器也能有桌面环境

### 结论

**向日葵 = 网络穿透层 + 视频流层 + 硬件控制层**
**remote_agent = 五感API层 + Shell层 + 文件层**

两者结合 = **任何网络环境下的完整五感远程控制**，且对本机零干扰（只读探索，未启动任何进程）。
