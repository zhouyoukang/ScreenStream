# 手机控电脑 — 全景分析

> **视角**：用户只有一部手机，世界在掌心6寸屏幕里。
> PC是"远方的身体"——眼睛看不到、手摸不着、耳朵听不见。
> 一切设计的终极目标：**让手机成为PC的延伸神经系统**。

---

## 一、用户的五感缺失模型

手机用户远程控PC时，面临的不是"功能缺失"而是**感官剥夺**：

| 感官 | PC前的体验 | 手机上的体验 | 痛感 |
|------|-----------|-------------|------|
| **视觉** | 27寸屏幕，一目十行 | 6寸屏，字小如蚁 | 🔴 最痛 |
| **触觉** | 鼠标精确到像素，键盘盲打 | 手指粗如柱，虚拟键盘慢 | 🔴 最痛 |
| **听觉** | 音箱/耳机实时 | 延迟/无音频 | 🟡 中等 |
| **认知** | 多窗口并行，一眼全局 | 单窗口，上下文丢失 | 🟡 中等 |
| **操控** | 快捷键飞速 | 每步都要点击 | 🔴 最痛 |

**核心洞察**：所有"手机控电脑"方案，本质上都是在**补偿这五种感官缺失**。方案优劣 = 补偿完整度 × 操作效率。

## 二、全市场方案扫描（12个方案，去杂后精华）

### A. 远程桌面类（看到PC屏幕 + 操控）

| 方案 | 类型 | 核心能力 | 延迟 | 手机体验 | 开源 |
|------|------|---------|------|---------|------|
| **RustDesk** | 自托管远控 | 屏幕镜像+键鼠+文件+剪贴板+WoL | 30-100ms | ⭐⭐⭐⭐ | ✅ |
| **Sunshine+Moonlight** | 游戏串流 | GPU硬编码H265/AV1+手柄+低延迟 | 5-20ms | ⭐⭐⭐⭐⭐ | ✅ |
| **Parsec** | 游戏串流 | 4:4:4色彩+手柄+多人 | 5-15ms | ⭐⭐⭐⭐⭐ | ❌ |
| Chrome Remote Desktop | 远控 | 零配置+Google账号 | 50-200ms | ⭐⭐⭐ | ❌ |
| TeamViewer/AnyDesk | 商业远控 | 跨平台+文件+会议 | 30-100ms | ⭐⭐⭐⭐ | ❌ |
| **你的remote_agent** | 自建HTTP | 截屏+键鼠+文件+Shell+服务管理 | 100-300ms | ⭐⭐⭐ | ✅ 自有 |

**去杂结论**：
- 🏆 **低延迟视觉**：Sunshine+Moonlight 碾压一切（GPU硬编码，5ms级）
- 🏆 **自主可控**：RustDesk（自托管relay）或你的remote_agent（零依赖）
- ❌ 淘汰：Chrome Remote Desktop（延迟高+依赖Google）、TeamViewer（商业限制+隐私）

### B. 遥控器类（不看PC屏幕，只发指令）

| 方案 | 核心能力 | 协议 | 手机体验 | 开源 |
|------|---------|------|---------|------|
| **Unified Remote** | 90+预设(媒体/PPT/鼠标/键盘) | WiFi/蓝牙 | ⭐⭐⭐⭐⭐ | ❌ |
| **KDE Connect** | 剪贴板/通知/文件/鼠标/媒体/命令 | TCP加密 | ⭐⭐⭐⭐ | ✅ |
| Macro Deck | 自定义宏按钮+Stream Deck替代 | WiFi | ⭐⭐⭐⭐ | ✅ |
| **你的remote_agent** | 35+ API(键鼠/文件/Shell/电源/服务) | HTTP | ⭐⭐⭐ | ✅ 自有 |

**去杂结论**：
- 🏆 **日常遥控之王**：KDE Connect（开源+通知同步+文件双向+剪贴板同步+媒体控制）
- 🏆 **极客定制之王**：Macro Deck / 你的remote_agent（可编程API）
- ❌ 淘汰：Unified Remote（闭源+订阅制+功能可替代）

### C. 协同类（手机与PC融合）

| 方案 | 核心能力 | 双向 | 开源 |
|------|---------|------|------|
| **KDE Connect** | 通知镜像+SMS回复+剪贴板双向+找手机+幻灯片遥控 | ✅ | ✅ |
| Barrier/Input Leap | 键鼠共享（手机当触控板） | 部分 | ✅ |
| Windows Phone Link | 通知+照片+通话+短信+APP镜像 | 部分 | ❌ |
| SpaceDesk | 手机当PC副屏 | 单向 | ❌ |

**去杂结论**：
- 🏆 **协同之王**：KDE Connect（唯一真正开源的双向融合方案）
- SpaceDesk 副屏有用但场景窄
- Phone Link 仅限Samsung/Surface联动

### 前端（remote_desktop.html，2201行，8面板）

Win(截屏+触摸) | Keys(快捷键) | Files(文件管理) | Proc(进程) | Clip(剪贴板) | Shell(终端) | Sys(系统/电源/服务) | Sess(跨会话)

### 独有优势（竞品没有的）
1. **MouseGuard** — 检测用户活动，自动暂停远程操控（防劫持）
2. **跨Windows会话** — 多账号部署+PSRemoting
3. **零依赖** — 纯Python stdlib + pyautogui，无框架
4. **完全可编程** — 所有功能都是HTTP API，AI Agent可直接调用

### 现有短板（对比竞品）

| 短板 | 竞品做法 | 影响 |
|------|---------|------|
| **截屏轮询，非实时流** | Sunshine: GPU H265硬编码实时流 | 🔴 视觉延迟大 |
| **无音频** | RustDesk: Opus音频流 | 🔴 听觉完全缺失 |
| **无通知同步** | KDE Connect: 双向通知镜像 | 🟡 认知缺口 |
| **无Wake-on-LAN** | RustDesk/Wolow: 魔术包唤醒 | 🟡 冷启动无解 |
| **无剪贴板自动同步** | KDE Connect: 实时双向 | 🟡 需手动操作 |
| **触控板模式粗糙** | Unified Remote: 多指手势+惯性滚动 | 🟡 精确度不足 |
| **无公网穿透内置** | RustDesk: relay中继 / Cloudflare Tunnel | 🟡 需外部配置 |

## 四、用户真实场景与痛点（以一部手机看世界）

### 场景1：躺在床上，PC在书桌上
**需求**：不想起身，想用手机操控PC看视频/听音乐/下载东西
- ✅ 已有：截屏看PC、点击操控、音量调节
- ❌ 缺失：**PC音频转发到手机**、**触控板模式太粗糙**

### 场景2：出门在外，PC在家
**需求**：远程查看PC状态、下载文件、执行脚本
- ✅ 已有：Shell执行、文件管理、系统信息
- ❌ 缺失：**公网穿透**（需Cloudflare/FRP）、**WoL远程开机**、**断线重连**

### 场景3：上课/开会，需要看PC上的资料
**需求**：快速调出PC上的文件或笔记
- ✅ 已有：文件浏览+下载
- ❌ 缺失：**搜索功能**、**文件预览**（PDF/图片直接看）

### 场景4：PC在做长任务（编译/下载/渲染）
**需求**：手机上看进度、完成后通知
- ✅ 已有：进程列表、Shell查状态
- ❌ 缺失：**主动推送通知到手机**、**任务完成提醒**

### 场景5：家人需要帮忙操作电脑
**需求**：远程帮家人解决电脑问题
- ✅ 已有：截屏+操控+Shell
- ❌ 缺失：**语音通话**、**标注画笔**、**手势引导**

### 场景6：手机当PC触控板/键盘（面对面）
**需求**：沙发上用手机当无线鼠标/键盘
- ✅ 已有：基础点击/打字
- ❌ 缺失：**触控板模式**（相对移动+手势）、**虚拟键盘优化**

## 六、推荐策略：三层架构

> **核心原则**：不重复造轮子。低延迟视频流让Sunshine做，双向协同让KDE Connect做，你的remote_agent专注"可编程深度控制"。

### 层1：借用成熟方案（零开发）

| 需求 | 方案 | 安装方式 |
|------|------|---------|
| 低延迟视频流 | **Sunshine** (PC端) + **Moonlight** (手机端) | winget install LizardByte.Sunshine |
| 双向通知+剪贴板+文件 | **KDE Connect** (PC+手机) | winget install KDEConnect |
| 公网穿透 | **Cloudflare Tunnel** (已有脚本) | 现有 remote-tunnel-setup.ps1 |

### 层2：增强 remote_agent（少量开发，高价值）

优先级从高到低：

#### P0 — 触控板模式（~50行JS）
```
前端新增"Touchpad"区域：
- 单指滑动 = 鼠标相对移动（不是绝对坐标点击）
- 双指滑动 = 滚轮
- 单指轻触 = 左键点击
- 双指轻触 = 右键
- 三指滑动 = 窗口切换（Alt+Tab）
- 长按拖动 = 拖拽
后端新增 POST /mouse/relative {"dx": 10, "dy": -5}
```

#### P1 — 通知推送（~40行Python）
```
后端新增：
- POST /watch {"cmd": "...", "notify_when": "done"}  — 监控命令完成
- GET /events (SSE长连接) — Server-Sent Events推送到手机
前端：Notification API + 振动 + 声音
```

#### P2 — Wake-on-LAN（~30行Python）
```
后端新增：
- POST /wol {"mac": "AA:BB:CC:DD:EE:FF"}
- 发送魔术包(UDP 255.255.255.255:9)
前端：Power面板增加"远程开机"按钮 + MAC地址配置
```

#### P3 — 快捷动作面板（~60行HTML/JS）
```
前端新增全屏网格面板（手机竖屏优化）：
🔒 锁屏 | 💤 睡眠 | 🔊 音量+ | 🔇 静音
⏯ 播放暂停 | ⏭ 下一曲 | 📸 截图 | 📋 剪贴板
🔍 搜索文件 | 📁 下载文件夹 | 💻 Shell | ⚡ 重启
每个按钮 = 一次HTTP调用，大按钮大文字适合手机竖屏单手操作
```

#### P4 — 文件搜索（~40行Python）
```
后端新增：
- GET /files/search?q=论文&path=C:\Users — 递归搜索
- 可选集成 Everything HTTP API (如果安装了)
前端：搜索栏 + 结果列表 + 一键下载
```

#### P5 — 剪贴板WebSocket双向同步（~60行）
```
后端新增 WebSocket /ws/clipboard：
- PC剪贴板变化 → 推送到手机
- 手机复制 → 推送到PC
- 500ms轮询检测变化
```

### 层3：长期演进（可选）

| 方向 | 方案 | 说明 |
|------|------|------|
| 音频流 | WebSocket + Opus编码 | ScreenStream已有AudioPlayer实现可复用 |
| AI语音控制 | Whisper STT → 命令解析 | "帮我打开论文文件夹" → /files?path=... |
| 手势宏 | 自定义手势 → 命令序列 | 画圈=锁屏，画V=音量+ |
| 统一控制中心 | remote_agent + ScreenStream + SmartHome 三合一 | 手机一个页面控制一切 |

> **Sunshine 给你眼睛，KDE Connect 给你双向感知，remote_agent 给你深入骨髓的控制力。三者组合 = 手机上的完整PC。**
