### 🥇 档位二：¥38/年 稳定方案（推荐！）

> **穿透**: FRP + 阿里云轻量服务器 | **控制**: remote_agent + RDP双通道

```
你的手机/外网电脑
    │
    ├── HTTP API → 阿里云ECS(FRP) → remote_agent :9903  (认知/管控)
    └── RDP客户端 → 阿里云ECS(FRP) → Windows RDP :3389  (视觉/听觉)
```

**为什么是¥38/年**：
- 阿里云轻量应用服务器 2核2G **200M带宽** 秒杀价 ¥38/年
- 不限流量，国内节点延迟20-40ms
- 同时跑FRP中继 + 可选RustDesk中继 / Tailscale DERP

**搭建步骤**：

**Step 1: 买阿里云轻量服务器**
- 访问 https://www.aliyun.com/product/swas
- 选 2核2G Ubuntu，**¥38/年秒杀**（注意选国内节点如杭州/上海）
- 安全组开放端口: 7000(FRP绑定), 7500(FRP控制台), 19903(remote_agent映射), 13389(RDP映射)

**Step 2: 服务器装FRP Server**
```bash
# SSH登录阿里云
ssh root@你的ECS公网IP

# 安装frps
wget https://github.com/fatedier/frp/releases/download/v0.61.1/frp_0.61.1_linux_amd64.tar.gz
tar xzf frp_0.61.1_linux_amd64.tar.gz
cd frp_0.61.1_linux_amd64

# 配置 frps.toml
cat > frps.toml << 'EOF'
bindPort = 7000
webServer.addr = "0.0.0.0"
webServer.port = 7500
webServer.user = "admin"
webServer.password = "你的控制台密码"
auth.method = "token"
auth.token = "你的FRP认证密码"
EOF

# 启动（用systemd守护）
sudo cat > /etc/systemd/system/frps.service << 'EOF'
[Unit]
Description=FRP Server
After=network.target

[Service]
Type=simple
ExecStart=/root/frp_0.61.1_linux_amd64/frps -c /root/frp_0.61.1_linux_amd64/frps.toml
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl enable frps
sudo systemctl start frps
```

**Step 3: 家里电脑装FRP Client**
```powershell
# 下载Windows版FRP
# https://github.com/fatedier/frp/releases → frp_0.61.1_windows_amd64.zip

# 配置 frpc.toml
@"
serverAddr = "你的ECS公网IP"
serverPort = 7000
auth.method = "token"
auth.token = "你的FRP认证密码"

[[proxies]]
name = "remote_agent"
type = "tcp"
localIP = "127.0.0.1"
localPort = 9903
remotePort = 19903

[[proxies]]
name = "rdp"
type = "tcp"
localIP = "127.0.0.1"
localPort = 3389
remotePort = 13389
"@ | Set-Content frpc.toml -Encoding UTF8

# 启动
.\frpc.exe -c frpc.toml
```

**Step 4: 远程访问**
```
# remote_agent (手机浏览器)
http://你的ECS公网IP:19903/?token=你的密码

# RDP (微软远程桌面APP)
你的ECS公网IP:13389
```

**五感体验**：
| 感官 | 通道 | 体验 | 评分 |

| 👁 视觉 | RDP视频流 | 60fps流畅桌面 | ⭐⭐⭐⭐⭐ |

> **穿透**: Tailscale + 阿里云自建DERP | **控制**: RDP + remote_agent
> **特点**: P2P直连时延迟仅5ms，比任何中转方案都快

```
你的手机/电脑 (Tailscale客户端)
    │
    ├── P2P直连 (NAT打洞成功, 5ms) ──→ 家里电脑 (Tailscale客户端)
    │                                      ├── remote_agent :9903
    │                                      └── RDP :3389
    │
    └── DERP中继 (打洞失败时, 10-30ms) ──→ 阿里云ECS (自建DERP)
                                              └── 中转到家里电脑
```

**额外步骤**（在档位二的基础上）：
```bash
# 阿里云ECS上部署DERP中继
docker run -d --restart always \
  -p 443:443 -p 3478:3478/udp \
  -e DERP_CERT_MODE=manual \
  -e DERP_DOMAIN=derp.你的域名.com \
  ghcr.io/yangchuansheng/derp:latest

# 家里电脑和手机都装Tailscale
# https://tailscale.com/download
# 配置ACL指向自建DERP
```

**比档位二多了什么**：
- P2P直连成功时：5ms延迟，满速上行（你家宽带上行速度）
- 失败时：走阿里云DERP中继，10-30ms（和FRP差不多）
- WireGuard端到端加密，比FRP更安全

## 六、解惑清单（所有之惑）

### Q1: "我应该选哪个？"

| 你的情况 | 推荐 |
|----------|------|
| **今天就想用，不想花钱** | 档位一（Cloudflare Quick Tunnel） |
| **稳定长期使用** | 档位二（FRP + 阿里云¥38/年） ⭐推荐 |
| **追求极致低延迟** | 档位三（Tailscale + 阿里云DERP） |
| **纯手机操控** | 档位一或二，用remote_agent的Web前端 |
| **写代码/看视频等视觉操作** | 必须加RDP通道（档位二/三） |
| **AI Agent自动化** | remote_agent是唯一选择，任何档位都行 |

### Q2: "Cloudflare Tunnel国内慢吗？"

**实话**：看运气。
- Cloudflare在国内无节点，走香港/日本/美国，延迟50-300ms不等
- 晚高峰可能更慢
- 但 **Quick Tunnel零成本零配置**，作为应急方案无敌
- 长期使用建议走档位二（国内ECS中转，延迟稳定20-40ms）

### Q3: "向日葵还要不要用？"

**结论**：保留但降级为应急工具。

| 向日葵保留的场景 | 原因 |
|-----------------|------|
| 电脑关机了，远程开机 | 唯一能WoL+智能插座唤醒的 |
| 要黑屏被控（旁人看不到） | 驱动级黑屏，无替代 |
| FRP/Tailscale全挂了 | 向日葵P2P穿透是兜底 |

其他场景全部用 remote_agent + FRP/Tailscale 替代。

### Q4: "RustDesk值得自建吗？"

**结论**：如果已经买了阿里云ECS做FRP，可以**顺手装RustDesk中继**（Docker一行命令），但 **不推荐把RustDesk当主力**。

原因：
- RustDesk的"认知层"（Shell/文件/进程/API）几乎为零
- 你已经有了 remote_agent 这个认知层怪兽
- RustDesk的价值 = 给不懂技术的家人一个简单远控入口

```bash
# 在阿里云ECS上顺手装RustDesk中继（可选）
docker run -d --name rustdesk-hbbs \
  -p 21115:21115 -p 21116:21116 -p 21116:21116/udp \
  -p 21118:21118 \
  --restart always \
  rustdesk/rustdesk-server hbbs

docker run -d --name rustdesk-hbbr \
  -p 21117:21117 \
  --restart always \
  rustdesk/rustdesk-server hbbr
```

### Q8: "电脑睡眠/关机了怎么办？"
