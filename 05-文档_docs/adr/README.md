# ADR 目录（架构决策记录）

> ADR 用于固定“为什么这样选”，避免后续反复争论与返工。

## 0) 命名规则

- `ADR-YYYYMMDD-<主题>.md`

## 1) 何时必须写 ADR

- 端口/入口/鉴权策略
- 对外 API/协议变更
- 合并策略（Quest/上游差异如何开关化）
- 影响多个模块的架构取舍

## 2) 模板（最小字段）

- 背景：现状 + 痛点（带证据路径）
- 现状证据：入口文件/关键类/默认配置
- 目标：一句话
- 决策：选择了什么
- 备选方案：至少 1 个
- 后果：影响面/风险/迁移策略
- 验收：如何确认决策正确
