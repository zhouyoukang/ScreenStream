# 宇树Go1电机通信测试

本目录包含用于测试与宇树Go1电机通信的各种脚本。以下是测试过程和结果的摘要。

## 测试环境

- 操作系统: Windows 10/11
- 测试设备: USB转RS485适配器
- 目标设备: 宇树科技Go1电机

## 检测到的串口

测试过程中，系统检测到以下串口：

1. COM5: USB-Enhanced-SERIAL CH343 - 可能是RS485适配器
2. COM1: 通信端口 (标准) - 系统自带
3. COM7: USB-SERIAL CH340 - 其他USB设备

## 测试结果

以下是进行的测试及其结果：

- **COM5端口测试**:
  - 状态: 无法访问（权限错误）
  - 原因: 可能被其他程序占用

- **COM7端口测试**:
  - 状态: 无法访问（权限错误）
  - 原因: 可能被其他程序占用

- **COM1端口测试**:
  - 状态: 可以访问
  - 波特率支持: 9600, 115200（不支持1000000及以上）
  - 通信测试: 成功发送命令，但未收到响应

## 可能的问题和解决方案

1. **COM端口被占用**
   - 关闭可能使用COM端口的其他程序
   - 在设备管理器中禁用并重新启用COM端口
   - 拔插USB设备后重新连接

2. **RS485连接问题**
   - 确认连接正确（A+/B-）
   - 检查电机电源（需要23-25V）
   - 验证适配器功能是否正常

3. **通信参数不匹配**
   - Go1电机通常使用5000000波特率，但测试中仅COM1可用，且只支持到115200
   - 可能需要使用支持更高波特率的串口适配器

4. **电机ID不正确**
   - 测试了ID 0, 1, 2, 4，但可能电机使用其他ID

## 主要测试脚本

1. `list_ports_only.py` - 列出可用串口
2. `fixed_hex_test.py` - 向电机发送硬编码命令
3. `com1_test.py` - 专门测试COM1端口
4. `listen_com1.py` - 在COM1上监听响应
5. `scan_all_ports.py` - 扫描所有端口、波特率、电机ID组合
6. `simple_twist.py` - 发送简单的电机扭动命令

## 结论

虽然我们能够成功建立与COM1端口的通信，但无法访问COM5和COM7（可能是连接电机的实际端口）。测试中发送的命令在COM1上没有收到响应，可能是因为COM1不是连接到电机的端口，或者通信参数不正确。

建议:
1. 重启计算机，关闭所有可能使用串口的应用程序
2. 使用设备管理器检查COM端口状态
3. 确认电机电源和RS485连接是否正确
4. 尝试使用官方提供的电机调试助手软件
