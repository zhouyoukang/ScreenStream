## 二、品牌详细适配

### 2.1 华为 / HarmonyOS

**无障碍服务设置路径：**
```
设置 → 辅助功能 → 无障碍 → 已安装的服务 → ScreenStream → 开启
```

**后台保活设置：**
```
1. 设置 → 电池 → 启动管理 → ScreenStream → 手动管理
   → 开启"允许自启动"、"允许关联启动"、"允许后台活动"
2. 设置 → 电池 → 更多电池设置 → 休眠时始终保持网络连接 → 开启
3. 多任务界面 → ScreenStream卡片 → 下拉锁定
```

**已知问题：**
- HarmonyOS可能在48小时后自动关闭无障碍服务
- 解决：开启"受保护的应用"或定期检查

---

### 2.2 小米 / MIUI / HyperOS

**无障碍服务设置路径：**
```
设置 → 更多设置 → 无障碍 → 已安装的服务 → ScreenStream → 开启
```

**后台保活设置：**
```
1. 设置 → 应用设置 → 应用管理 → ScreenStream → 自启动 → 开启
2. 设置 → 电量和性能 → 电量 → ScreenStream → 无限制
3. 安全中心 → 应用管理 → 权限 → ScreenStream → 允许后台弹出界面
4. 多任务界面 → ScreenStream卡片 → 长按 → 锁定
```

**已知问题：**
- MIUI开启无障碍时弹二次确认"是否信任此服务"
- HyperOS（小米14+）无障碍路径可能变更
- 省电模式下可能杀后台

---

### 2.3 OPPO / ColorOS

**无障碍服务设置路径：**
```
设置 → 系统设置 → 无障碍 → 已安装的服务 → ScreenStream → 开启
```

**后台保活设置：**
```
1. 设置 → 电池 → 更多电池设置 → 优化电池使用 → ScreenStream → 不优化
2. 设置 → 应用管理 → ScreenStream → 省电策略 → 允许后台运行
3. 多任务界面 → ScreenStream → 下拉锁定
```

**已知问题：**
- ColorOS "智能冻结"可能冻结后台服务
- 解决：关闭应用冻结功能

---

### 2.6 三星 / OneUI

**无障碍服务设置路径：**

## 三、通用问题与解决方案

### 3.1 MediaProjection弹窗

**问题：** 每次启动投屏，Android系统会弹出"开始录制屏幕"确认弹窗。

**解决：**
- Android 14+：系统级弹窗无法跳过，只能引导用户点确认
- 考虑添加Shizuku支持，通过一次性ADB授权后自动确认

### 3.2 无障碍服务被关闭

**问题：** 部分品牌在重启、更新、或电池优化后关闭无障碍服务。

**解决方案（优先级递减）：**
1. APP检测A11y状态，未开启时弹出引导弹窗
2. Shizuku API自动重新启用（需一次性授权）
3. 引导用户手动重新开启

### 3.3 Android 13+ 限制

**问题：** Android 13限制后台启动前台服务、限制通知权限。

**解决：**
- 请求POST_NOTIFICATIONS权限
- 使用FOREGROUND_SERVICE_MEDIA_PROJECTION类型

### 3.4 Android 14+ 限制

**问题：** Android 14限制隐式Intent、更严格的前台服务类型。

**解决：**
- 声明 `foregroundServiceType="mediaProjection"`
- 使用显式Intent

## 四、Shizuku集成方案（推荐增强）

Shizuku可大幅降低用户配置难度：

| 功能 | 无Shizuku | 有Shizuku |
|------|-----------|-----------|
| 开启无障碍服务 | 用户手动设置 | 自动开启 |
| MediaProjection确认 | 每次弹窗 | 可自动确认(Android 14以下) |
| 输入注入 | AccessibilityService | 可选ADB级别注入 |

**Shizuku获取方式：**
1. 用户安装Shizuku APP
2. 通过开发者选项启用"无线调试"
3. 在Shizuku中配对一次即可
4. 之后ScreenStream可自动获取权限
