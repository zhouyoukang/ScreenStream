## 二、四层架构设计

### Layer 1: 采集与编码层（手机端）

```kotlin
MediaProjection
    → VirtualDisplay (屏幕镜像)
    → Surface (帧缓冲)
    → MediaCodec H264/H265 (硬件编码)
    → WebRTC VideoTrack (P2P传输)
    ↘ BitmapCapture (MJPEG/WebSocket降级)
```

**已有实现**：`BitmapCapture.kt` + `WebRtcP2PClient.kt`

### Layer 2: 传输层（多路径）

| 路径 | 协议 | 入口 | 优先级 |
|------|------|------|--------|
| **WebRTC P2P** | UDP DTLS-SRTP | DataChannel + VideoTrack | P0（首选） |
| **FRP H264 WS** | WebSocket over TCP | wss://aiotvr.xyz/screen/stream/h264 | P1（降级） |
| **FRP MJPEG** | HTTP long-polling | https://aiotvr.xyz/screen/stream/mjpeg | P2（保底） |

### Layer 3: 信令层（服务器端）

```
信令服务器 (aiotvr.xyz:9100)
    ├── /signal/ WebSocket — P2P房间协商
    │   ├── provider注册（手机端）
    │   ├── viewer加入（浏览器端）
    │   ├── SDP Offer/Answer交换
    │   └── ICE Candidate交换
    └── /api/status — 健康检查
```

### Layer 4: 控制层（双向）

```
浏览器触控事件
    → WebRTC DataChannel (JSON消息)
    → 手机端 WebRtcP2PClient.forwardToInput()
    → HTTP localhost:8084 (InputRoutes.kt)
    → InputService (AccessibilityService)
    → Android系统
```

## 三、信令协议设计

### 3.1 消息格式

```json
// 手机端 → 信令服务器：注册为provider
{
  "type": "register",
  "role": "provider",
  "room": "123456",
  "device": "OnePlus Nord CE3",
  "auth": "sha256(room+timestamp+secret)"
}

// 浏览器端 → 信令服务器：加入为viewer
{
  "type": "join",
  "role": "viewer",
  "room": "123456"
}

// 信令服务器 → 浏览器：provider的offer
{
  "type": "offer",
  "data": { "sdp": "v=0\r\n...", "type": "offer" },
  "_viewerId": "viewer_abc123"
}

// 浏览器 → 信令服务器：viewer的answer
{
  "type": "answer",
  "data": { "sdp": "v=0\r\n...", "type": "answer" },
  "_viewerId": "viewer_abc123"
}

// 双向：ICE candidate
{
  "type": "candidate",
  "data": {
    "candidate": "candidate:...",
    "sdpMid": "0",
    "sdpMLineIndex": 0
  },
  "_viewerId": "viewer_abc123"
}
```

### 3.2 连接流程

```
时间线 →

父母手机                    信令服务器                    子女浏览器
   │                          │                            │
   │── register(provider) ───→│                            │
   │←── registered ──────────│                            │
   │                          │                            │
   │                          │←── join(viewer) ──────────│
   │←── viewer_joined ───────│── viewer_joined ──────────→│
   │                          │                            │
   │── offer(SDP) ──────────→│── offer(SDP) ─────────────→│
   │                          │                            │
   │                          │←── answer(SDP) ───────────│
   │←── answer(SDP) ─────────│                            │
   │                          │                            │
   │── candidate(ICE) ──────→│── candidate(ICE) ─────────→│
   │                          │←── candidate(ICE) ────────│
   │←── candidate(ICE) ──────│                            │
```

## 四、多方案矩阵

### 4.1 方案A：WebRTC P2P（首选）

```
适用条件：双方网络支持UDP，STUN打洞成功
延迟：50-200ms
画质：H264 1.5Mbps（可调）
控制：DataChannel，延迟<50ms
成本：¥0（STUN免费）
```

### 4.2 方案B：WebRTC + TURN中继

```
适用条件：P2P失败，对称NAT等
延迟：100-300ms
画质：同方案A
控制：同方案A
成本：TURN流量（metered.ca免费额度/自建）
```

### 4.3 方案C：FRP WebSocket穿透

```
适用条件：WebRTC完全失败
延迟：100-400ms（取决于FRP服务器位置）
画质：H264 WebSocket
控制：HTTP API via FRP反代
成本：FRP服务器（已有，¥38/年）
```

### 4.4 方案自动选择逻辑

```javascript
async function connectToParent(roomCode) {
  // 1. 尝试 WebRTC P2P
  try {
    const pc = await connectWebRTC(roomCode);
    if (pc.iceConnectionState === 'connected') {
      showStatus('P2P直连成功，最佳体验');
      return { mode: 'webrtc-p2p', pc };
    }
  } catch (e) { console.log('WebRTC P2P failed:', e); }

  // 2. 降级到 FRP WebSocket
  try {
    const ws = await connectFRPWebSocket();
    showStatus('通过中继连接，体验良好');
    return { mode: 'frp-ws', ws };
  } catch (e) { console.log('FRP WS failed:', e); }

  // 3. 最后降级到 MJPEG
  showStatus('基础模式连接');
  return { mode: 'mjpeg', url: getMjpegUrl() };
}
```

## 五、安全架构

### 5.1 房间码机制

```
1. 父母手机开启投屏 → 生成6位随机房间码（SecureRandom）
2. 房间码通过以下方式传递给子女：
   a. APP内显示 → 电话/微信告知子女
   b. 生成QR码 → 微信扫描
   c. 生成链接 → 微信/短信发送
3. 链接格式：https://aiotvr.xyz/cast/?room=123456
4. 房间码有效期：默认24小时，可配置
```

### 5.2 多层鉴权

```
Layer 1: 房间码 — 进入房间的门票
Layer 2: DTLS-SRTP — WebRTC端到端加密
Layer 3: Bearer Token — API鉴权（降级模式）
Layer 4: HTTPS — 信令通道加密
```
