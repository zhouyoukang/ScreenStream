### v4 Agent API + 双模式增强 (5项)

#### ✅ FIX-13: relay-server Agent REST API — 新增

**位置**: `relay-server/server.js` 新增 ~180行
**功能**: 为Agent提供HTTP端点，可程序化控制手机，无需浏览器。

| 端点 | 方法 | 功能 |
|------|:----:|------|
| `/api/rooms` | GET | 列出所有在线房间+设备信息 |
| `/api/room/:id` | GET | 单个房间详情(设备/统计/连接状态) |
| `/api/command` | POST | 发送控制命令到手机(触摸/按键/文本/控制) |
| `/api/batch` | POST | 批量发送多条命令(支持间隔延迟) |
| `/api/status` | GET | 服务状态(含 pendingRequests) |

**鉴权**: `X-Token` header 或 `?token=` 参数 或 `Authorization: Bearer`
**响应模式**: 火并忘(fire-and-forget)，命令已发送但手机端APP未回复`_reqId`确认

#### ✅ FIX-14: signaling-server CORS头 — 新增

**位置**: `signaling-server/server.js:24-28`
**问题**: 缺少CORS头，跨域前端无法调用API。
**修复**: 添加 `Access-Control-Allow-Origin: *` 等头。

#### ✅ FIX-15: setup.html 双模式(直连+中继) — 增强

**位置**: `viewer/setup.html`
**问题**: 仅支持FRP直连，FRP隧道不通时无法控制手机。
**修复**: 自动尝试直连→失败后降级到Relay Agent API→通过中继服务器控制手机。

#### ✅ FIX-16: viewer sendCmd无通道反馈 — 增强

**位置**: `viewer/index.html:657-668`
**问题**: `sendCmd`在所有发送通道不可用时无任何反馈。
**修复**: 添加console.warn和状态栏提示“无控制通道”。

### v3 高级工程师深度审计修复 (6项)

#### 🔴 FIX-7: relay-server路径穿越漏洞 — 已修复 ✅

**位置**: `relay-server/server.js:184-203`
**问题**: 静态文件服务直接用`url.pathname`拼接`path.join`，攻击者可构造`/../../../etc/passwd`读取任意文件。
**修复**: `path.resolve()` + `startsWith(viewerDir)` 白名单校验。

#### 🟡 FIX-8: relay-server启动日志泄露RELAY_TOKEN明文 — 已修复 ✅

**位置**: `relay-server/server.js:417`
**问题**: `log('Token: ' + TOKEN)` 将完整token写入日志/stdout。
**修复**: 仅显示前4字符 + 星号遮蔽: `Token: scre************`。

#### 🟡 FIX-9: relay-server同步IO阻塞事件循环 — 已修复 ✅

**位置**: `relay-server/server.js:200`
**问题**: `readFileSync` 在视频帧中继过程中阻塞Node.js事件循环，影响实时性能。
**修复**: 替换为 `createReadStream().pipe(res)` 异步流式传输。

#### 🟡 FIX-10: signaling-server依赖express(生产环境未安装) — 已修复 ✅

**位置**: `signaling-server/server.js`
**问题**: 使用express框架，但生产服务器node_modules仅有ws，导致服务crash-loop。
**修复**: 重写为raw `http.createServer`，与relay-server统一模式，消除express依赖。

#### 🟡 FIX-11: viewer控制命令路径不一致 — 已修复 ✅

**位置**: `viewer/index.html:186-187`
**问题**: `sendCtl('volume_up')` 下划线格式与InputRoutes.kt的 `/volume/up` 斜杠格式不一致，依赖Kotlin映射表转换。
**修复**: 统一为 `sendCtl('volume/up')`, `sendCtl('volume/down')`，直通InputService。

#### 🟢 FIX-12: viewer信令消息解析静默吞错 — 已修复 ✅

**位置**: `viewer/index.html:283`
**问题**: `catch(e){}` 空catch块，信令解析失败时无任何日志，难以排查问题。
**修复**: `catch(e){console.warn('[Signal] parse error',e)}`。

## 八、Agent API 使用手册

### 架构

```
Agent (PowerShell/Python)
  │ HTTP REST
  ↓
Relay Server (aiotvr.xyz/relay/api/*)
  │ WebSocket
  ↓
Android Phone (ScreenStream CloudRelayClient)
  │ InputService API
  ↓
AccessibilityService (70+ 操作)
```

### 端点详解

#### 1. 查看在线设备

```powershell
Invoke-RestMethod -Uri "https://aiotvr.xyz/relay/api/rooms" -Headers @{"X-Token"="screenstream_2026"}
```

#### 2. 发送单条命令

```powershell
# 触摸点击
$body = '{"room":"ROOM_ID","type":"touch","data":{"action":"tap","x":540,"y":1200}}'
# 按键 (home/back/recents)
$body = '{"room":"ROOM_ID","type":"control","data":{"action":"home"}}'
# 输入文本
$body = '{"room":"ROOM_ID","type":"text","data":{"text":"你好"}}'
# 音量控制
$body = '{"room":"ROOM_ID","type":"control","data":{"action":"volume/up"}}'
# 滑动
$body = '{"room":"ROOM_ID","type":"touch","data":{"action":"swipe","x1":540,"y1":1800,"x2":540,"y2":600,"duration":300}}'

Invoke-WebRequest -Uri "https://aiotvr.xyz/relay/api/command" -Method POST -Headers @{"X-Token"="screenstream_2026";"Content-Type"="application/json"} -Body $body -UseBasicParsing
```

#### 3. 批量命令

```powershell
$body = @{
  room = "ROOM_ID"
  commands = @(
    @{type="control"; data=@{action="home"}; delay=500},
    @{type="touch"; data=@{action="tap"; x=540; y=1200}; delay=1000},
    @{type="text"; data=@{text="hello"}}
  )
} | ConvertTo-Json -Depth 5
Invoke-WebRequest -Uri "https://aiotvr.xyz/relay/api/batch" -Method POST -Headers @{"X-Token"="screenstream_2026";"Content-Type"="application/json"} -Body $body -UseBasicParsing
```

### 当前限制
