🛠 问题与解决方案

18个关键问题，按严重度分类。点击展开详情。

#1 PowerShell Hooks 全局终端卡死
根因hooks.json 中 PowerShell 钩子干扰终端提示符检测
解决清除所有 PS hooks，立铁律禁止。Python/Node.js 安全可用
教训全局配置修改前必须：影响评估 → 备份 → 验证
#2 微信反无障碍三重阻止 (P21)
根因rootInActiveWindow / getWindows / 事件 全部 blocked
解决三层回退：getWindows → UsageStatsManager → event_cache
教训核心接口必须有降级路径
#3 force-stop 禁用 AccessibilityService (P23)
根因Android 系统在 force-stop 时自动从 enabled 列表移除
解决部署脚本自动 settings put 重新启用，已集成到 dev-deploy.ps1
#4 XSS 注入 (5轮修复)
范围8+ 面板动态数据未转义：宏/设备信息/通知/应用监控/WebSocket
解决统一 escapeHtml()，完整 &<>"' 转义
#5 路径穿越 (3处)
范围renameFile / saveFile / upload 参数含 /\.. 可写任意路径
解决拒绝含路径分隔符的参数 + sanitizePath 沙箱
#6 信息泄露 — HttpServer 异常堆栈
解决StatusPages 异常处理改为固定 "500: Internal Server Error"
#7 PIN 可预测 (kotlin.random)
解决替换为 java.security.SecureRandom
#8 媒体管线资源泄漏
范围BitmapCapture 每帧泄漏 / H264Encoder callback 异常导致停滞 / AudioStreamer 未释放
解决try-finally + recycle() + releaseOutputBuffer
#9 OPPO 安全拦截 APP 启动 (P24-P26)
现象com.oplus.securitypermission 弹窗拦截部分APP
解决monkey 命令走 shell 权限绕过。回退链：openapp → Intent MAIN → dismiss → 记录OEM限制
#10 剪贴板 Android 10+ 后台限制 (P22)
根因AccessibilityService 后台服务无法读取 primaryClip
解决写入时同步缓存到 cachedClipboardText，读取返回缓存值
#11 端口不固定 (P1)
解决扫描 8080-8099 找 /status 响应，dev-deploy.ps1 自动探测
#12 京东反自动化 (P28)
现象ADB input tap + AccessibilityService 触发安全验证
结论无法自动绕过。需人工完成一次后短期内不再触发
#13 拼多多搜索框定位 (P29)
根因原 tap 坐标偏低 20px
解决调整为 (570,170) 匹配实际 bounds 中心
#14 MiCloud 2FA 阻断
解决从 HA 缓存提取 xiaomiio session token，绕过 2FA 登录
#15 Windows 系统代理阻断 MiCloud
根因127.0.0.1:7897 代理阻断 account.xiaomi.com
解决NO_PROXY=account.xiaomi.com,api.io.mi.com
#16 isOnline 误判
根因GET code=0 只是缓存响应，非真实在线状态
解决改用 isOnline 字段精确判断设备实际状态
#17 错误码 -704030023 误读
真相property_read_only（属性只读），非 device_offline
解决新增错误码映射表
#18 理论过重实践不足 (Phone Agent)
现象25个文件、22个文档，但可用功能 0 个
教训验证 ≠ 交付；后端 ≠ 前端；文档 ≠ 功能
解决整合为 6 文件，四层金字塔验证全部通过

洞察 4

洞察 5
observe → think → act → verify 乃 AGI 操控手机的最小可运行架构。
洞察 6
音箱语音代理 = 远程控制万能钥匙。1台在线音箱穿透设备"离线"限制，控全屋。
洞察 7
最大化可达性，最小化依赖 — 四层金字塔是最优成本结构。

▲ 四层金字塔

AI是可选加速层，不是必要依赖。实测验证通过。

L3: 自主Agent — 监控循环+决策+自愈
覆盖 2% | 需大模型 | 5/5 Agent任务通过
L2: LLM推理 — 自然语言 → 操作序列
覆盖 8% | 需小模型 | /command 验证通过
L1: 组合序列 — 多步确定性流程
覆盖 20% | 零AI | 6/6 + 复杂联动 5/5 (43步)
L0: 原子API — 单个HTTP调用
覆盖 70% | 零AI | 29/29 = 100% | curl即可
L0+L1 覆盖 90% 日常需求（零成本） · 35/36项核心功能脱离AI完全可用
