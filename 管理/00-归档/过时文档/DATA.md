# ScreenStream_v2 数据管理哲学（DATA）

> 版本：v32+ | 更新：2026-02-21
> 定位：数据是系统的血液，本文定义数据如何流动、存储、保护和演进。

## 核心哲学

**数据三原则**：
- **可追溯** — 每一条数据都有来源、时间戳和生命周期
- **不散落** — 同类数据只有一个权威存储位置，禁止多处重复
- **可恢复** — 任何状态变更都可回退到已知安全点

**深层认知**：数据管理不是"把文件放整齐"，而是建立一套**信息流动的引力场**——
让正确的数据在正确的时刻流向正确的消费者，同时确保错误数据无法扩散。

## 0) 数据分类体系

| 层级 | 类别 | 生命周期 | 存储位置 | 敏感度 |
|------|------|---------|---------|--------|
| L0 | **静态资产** | 永久 | Git 仓库 | 公开 |
| L1 | **配置数据** | 持久 | SharedPreferences / DataStore | 中 |
| L2 | **运行时状态** | 会话级 | 内存 (Kotlin object) | 低 |
| L3 | **诊断数据** | 短期 | XLog / Logcat | 中 |
| L4 | **用户数据** | 用户控制 | 设备存储 / 剪贴板 | 高 |
| L5 | **秘密数据** | 永不暴露 | Keystore / .env | 极高 |

### L0: 静态资产（代码即资产）

资产是系统运行的"骨架"，变更频率低但影响面大。

| 资产 | 位置 | 变更规则 |
|------|------|---------|
| Web UI | `mjpeg/assets/index.html` | 前后端同步，编译后生效 |
| JS 依赖 | `mjpeg/assets/jmuxer.min.js` | 版本锁定，升级需测试 |
| 图标/证书 | `res/` / `keystore/` | keystore 绝不入 Git |
| SQL Schema | 未来 Room | 版本化迁移（Migration） |

### L1: 配置数据（系统的"性格"）

配置决定系统的行为边界。**配置变更 = 行为变更**，必须像对待代码一样对待。

```
配置层次（优先级从高到低）：
用户实时修改 > SharedPreferences 持久化 > 代码默认值 > 编译时常量
```

| 配置域 | 关键项 | 文件 | 变更影响 |
|--------|-------|------|---------|
| 全局 | `streamingStartsOnBoot` | `AppSettings.kt` | 开机行为 |
| MJPEG | `serverPort`, `jpegQuality` | `MjpegSettings` | 端口/画质 |
| RTSP | `rtspPort` | `RtspSettings` | RTSP 端口 |
| WebRTC | `webrtcPort` | `WebRtcSettings` | WebRTC 端口 |
| Input | `apiPort`, `requirePin`, `inputEnabled` | `InputSettings.kt` | 安全/控制 |
| 宏系统 | 宏定义 JSON | `MacroEngine.kt` (内存) | 自动化行为 |

**铁律**：端口/入口/鉴权配置变更 → 必须先写 ADR (`05-文档_docs/adr/`)

### L2: 运行时状态（系统的"脉搏"）

运行时状态反映系统此刻的健康度。它是**瞬态的、易失的**，但却是诊断问题的第一现场。

| 状态 | 数据源 | 查询方式 | 消费者 |
|------|--------|---------|--------|
| 投屏中？ | MediaProjection | `/status` API | 前端状态栏 |
| 输入已连接？ | AccessibilityService | 服务绑定状态 | 前端控制面板 |
| 当前连接数 | HttpServer sessions | `/status` 扩展 | 监控仪表盘 |
| 前台应用 | AccessibilityEvent | `/foreground` API | AI Brain |
| View 树 | AccessibilityNodeInfo | `/viewtree` API | AI 决策引擎 |
| 通知缓存 | NotificationListener | `/notifications/read` | 通知中心面板 |
| 宏执行状态 | MacroEngine | `/macro/running` | 宏管理面板 |

**设计原则**：运行时状态不持久化（除非有明确需求）。重启即归零，这是特性不是缺陷。

### L3: 诊断数据（系统的"病历"）

```
诊断数据金字塔：
        /\
       /  \  崩溃堆栈（必须保存）
      /----\
     / XLog \  结构化日志（按需保存）
    /--------\
   / Logcat   \  实时流（不保存，按需查看）
  /------------\
 / 性能指标     \  FPS/延迟/内存（采样保存）
/________________\
```

| 级别 | 工具 | 保留策略 | 查看方式 |
|------|------|---------|---------|
| 崩溃 | 系统 tombstone | 永久 | `adb bugreport` |
| 错误 | XLog ERROR | 7天滚动 | 日志文件 |
| 警告 | XLog WARN | 3天滚动 | 日志文件 |
| 信息 | XLog INFO | 1天 | 日志文件 |
| 调试 | Logcat DEBUG | 不保存 | `adb logcat` 实时 |

### L4: 用户数据（用户的"财产"）

ScreenStream 处理用户数据的原则：**最小接触，最大尊重**。

| 数据 | 访问方式 | 权限 | 处理原则 |
|------|---------|------|---------|
| 屏幕内容 | MediaProjection | 用户显式授权 | 仅投屏传输，不存储 |
| 文件系统 | Storage API | Scoped Storage | 沙箱限制，不越界 |
| 剪贴板 | ClipboardManager | 前台时可读 | 读后不缓存 |
| 通知内容 | NotificationListener | 用户授权 | 内存缓存50条，不持久化 |
| 应用列表 | PackageManager | 自动获得 | 仅列表，不分析行为 |

### L5: 秘密数据（绝对禁区）

| 数据 | 存储 | 规则 |
|------|------|------|
| 签名密钥 | `keystore/` | .gitignore 永久排除 |
| PIN 码 | SharedPreferences (加密) | 明文绝不出现在日志 |
| API 密钥 | `.env` | .gitignore + 不硬编码 |
| 用户凭证 | Android Keystore | 硬件级保护 |

## 1) 数据流动拓扑

```
┌─────────────────────────────────────────────────────────┐
│                    数据流动全景                           │
│                                                         │
│  [屏幕像素] ──MediaProjection──→ [编码器] ──→ [投屏流]   │
│       │                                      ↓          │
│       │                              MJPEG / H264 / H265│
│       │                              WebRTC / RTSP      │
│       ↓                                      ↓          │
│  [View树] ──Accessibility──→ [AI Brain] ──→ [API响应]   │
│       │                          ↑                      │
│       │                     [宏引擎] ←── [用户规则]      │
│       ↓                          ↓                      │
│  [事件流] ──────────────→ [InputService] ──→ [设备操作]  │
│   触控/键盘/手势              ↑                          │
│                          [HTTP请求]                      │
│                          来自浏览器                       │
└─────────────────────────────────────────────────────────┘
```

### 数据流分类

| 流向 | 协议 | 延迟要求 | 可靠性要求 |
|------|------|---------|-----------|
| 屏幕→浏览器 | MJPEG/H264/WebRTC | <100ms | 允许丢帧 |
| 浏览器→设备 | HTTP REST | <200ms | 必须可靠 |
| 实时触控 | WebSocket | <50ms | 允许丢包 |
| AI 查询 | HTTP GET | <500ms | 必须可靠 |
| 宏执行 | 内部调用 | 无硬限制 | 必须可靠+有序 |

### 端口即边界

```
端口分配（固定，禁止冲突）：
  8080 — Gateway（网关路由）
  8081 — MJPEG（主入口，承载 Web UI + Input 路由）
  8082 — RTSP
  8083 — WebRTC
  8084 — Input（兼容入口）
```

每个端口 = 一个独立的数据边界。跨端口通信通过 Kotlin 内部接口，不走网络。

## 2) 数据一致性模型

### 单一事实源（Single Source of Truth）

```
❌ 错误模式：                    ✅ 正确模式：
  前端有一份端口配置                前端从 /status 读取
  后端有一份端口配置         →      后端从 Settings 读取
  文档有一份端口配置                文档从 Settings 引用
  三者经常不一致                    Settings = 唯一事实源
```

| 数据 | 事实源 | 消费者 |
|------|--------|--------|
| 端口配置 | `*Settings.kt` | HttpServer / 前端 / 文档 |
| API 路由 | `InputRoutes.kt` | 前端按钮 / 验证脚本 / 文档 |
| 功能清单 | `FEATURES.md` | README / STATUS / 外部引用 |
| 模块映射 | `settings.gradle.kts` | 文档 / CI / 构建脚本 |

### 状态同步策略

| 场景 | 策略 | 理由 |
|------|------|------|
| 配置变更 | 重启生效 | 简单可靠，无竞态 |
| 投屏状态 | 事件驱动 | MediaProjection 回调 |
| View 树 | 按需快照 | 避免性能开销 |
| 宏状态 | 内存同步 | 单线程执行，无冲突 |
| 通知缓存 | FIFO 50条 | 内存可控，自动淘汰 |

## 3) 数据生命周期管理

### 创建 → 使用 → 归档 → 销毁

```
数据生命线：
  创建 ──→ 活跃使用 ──→ 冷存储 ──→ 安全销毁
   │          │            │           │
   │          │            │           └── 日志轮转/缓存清理
   │          │            └── 旧日志压缩/旧宏归档
   │          └── API 响应/投屏传输/宏执行
   └── 用户操作/系统事件/API 请求
```

### 清理策略

| 数据类型 | 清理时机 | 方式 |
|---------|---------|------|
| 投屏帧 | 实时 | 编码后立即释放 Bitmap |
| View 树快照 | 每次查询 | 旧快照被新快照替换 |
| 通知缓存 | 超过50条 | FIFO 自动淘汰最旧 |
| 宏执行日志 | 宏引擎重启 | 内存清空 |
| XLog 文件 | 按天/大小 | 滚动删除 |
| 截图文件 | 用户手动 | 存储在标准 Pictures/ |

### 异常数据处理

| 异常 | 处理 | 原则 |
|------|------|------|
| 配置损坏 | 回退默认值 | 可用 > 精确 |
| 日志写满 | 滚动覆盖 | 新数据 > 旧数据 |
| 内存溢出 | 释放缓存 | 核心功能 > 辅助功能 |
| 网络中断 | 缓冲重连 | 静默恢复，不弹窗 |

## 4) 数据安全护栏

### 四层防御

```
┌─────────────────────────────────┐
│  Layer 4: 代码审查              │  ← 人工：敏感操作必须 Review
├─────────────────────────────────┤
│  Layer 3: .gitignore            │  ← 自动：秘密文件永不入库
├─────────────────────────────────┤
│  Layer 2: Android 沙箱          │  ← 系统：Scoped Storage 隔离
├─────────────────────────────────┤
│  Layer 1: 权限模型              │  ← 系统：用户显式授权
└─────────────────────────────────┘
```

### 不可违反的铁律

1. **keystore / 私钥 / .env** — `.gitignore` 永久排除，违反即安全事故
2. **PIN 码** — 明文绝不出现在日志、API 响应、前端代码中
3. **用户屏幕内容** — 仅投屏传输，不存储到服务器/云端
4. **配置变更影响端口/入口/鉴权** — 必须先写 ADR
5. **文件操作** — `sanitizePath()` 沙箱限制，不可访问 ExternalStorage 以外路径

## 5) 数据演进路线

### 当前（v32+）：内存为主，轻量持久化

```
配置 → SharedPreferences / DataStore（持久）
状态 → Kotlin object 单例（内存）
宏   → MacroEngine 内存存储（会话级）
日志 → XLog 文件（滚动）
```

### 中期：结构化存储

- 宏定义持久化（Room / JSON 文件）
- 执行历史持久化（SQLite）
- 用户偏好同步（DataStore Proto）

### 长期：云端协同

- 宏/规则跨设备同步
- 执行日志云端聚合
- 设备状态远程监控

## 附：数据问题诊断清单

遇到数据相关问题时，按此顺序排查：

| # | 检查项 | 命令/方法 | 期望 |
|---|--------|---------|------|
| 1 | 端口监听？ | `adb forward` + `curl /status` | 200 OK |
| 2 | 配置正确？ | 查看 Settings 对应 DataStore | 值符合预期 |
| 3 | 日志有错？ | `adb logcat -s ScreenStream` | 无 ERROR |
| 4 | 权限已授？ | 系统设置→无障碍/通知 | 已开启 |
| 5 | 数据可达？ | 对应 API GET | JSON 非空 |
| 6 | 状态一致？ | 前端显示 vs API 返回 | 一致 |
