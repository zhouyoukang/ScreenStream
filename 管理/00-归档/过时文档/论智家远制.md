# 论智家远制

> 岁在丙午，仲春之望，余居于外，欲远操家中万器。遂以三路并进，终得其法。兹以文言统摄所成，俾后来者览之知要。

---

## 一、立意

夫智家之制，其要在**通**。
通者，人居于外，而家中灯火、音器、风扇、帘幕，皆可如臂使指也。

然国内之厂，各设壁垒：
- **小米**据六亿设备，而不开API于个人
- **涂鸦**最为开放，然体验版有量之限
- **易微联**开关插座称王，需注册开发者
- **华为鸿蒙**最为封闭，唯ScreenStream可破之

余乃思：**不必一一攻城，但得一枢纽而统之**。

---

## 二、三径

### 径一：MiCloud 直通（取其凭而用之）

```
HA 缓存 → session_token(xiaomiio) → api.io.mi.com → 24 台设备
```

HA（Home Assistant）既已登录米家，其凭据藏于 `.storage/core.config_entries`。
取 `user_id`、`service_token`、`ssecurity` 三者，即可直调 MiCloud RPC。

**实测**：十三功能皆通——音量、TTS、AI指令、播放控制。
**要害**：设备须 `isOnline=True`，二十四台中仅一台在线。

---

### 径二：音箱代理（借一箱而令万器）

```
远程 → MiCloud → 在线音箱 → execute_command("打开灯带") → 家庭内万器
```

此为本次**最大突破**。

虽二十三台设备显示离线（实为家中路由器/互联网断线，设备本身在家庭WiFi内仍活），
然音箱一台在线，即可发语音指令，由其在家中局域网内代为执行。

**实测**：十七条指令皆通——灯带明灭、筒灯调色、风扇启停、音箱播报。
**妙处**：不依赖设备云端在线，只要音箱活着，万器皆可控。
**喻之**：音箱者，家中之管家也。主人在外传书一封，管家自去张灯结彩。

---

### 径三：Mina API（取小爱之脑而用之）

```
Playwright 登录 → passToken → serviceLogin(micoapi) → serviceToken → api2.mina.mi.com
```

此乃 MIGPT-Easy 所用之**未公开API**，与 MIoT 协议并行而不同：
- MIoT 者，设备属性之读写也（SIID/PIID）
- Mina 者，音箱大脑之调度也（TTS/对话/播放）

**难关**：小米 SSO 以 SID 区分服务。HA 缓存者 `xiaomiio` 也，Mina 需 `micoapi` 也。
二者 serviceToken 不可混用。

**破局**：以 Playwright 驱动浏览器，自动填入手机号与密码，勾选协议，点击登录。
小米未触发验证码（或因浏览器指纹正常），遂登录成功。
再从 cookies 取 passToken，换得 micoapi 之 serviceToken。

**实测**：五台 Mina 设备发现，一台 TTS 成功。

---

## 三、枢纽

三径之上，设 **Gateway** 以统之：

```
            ┌─ MiCloud MIoT ──→ 24 台设备属性
Gateway ────┼─ 音箱语音代理 ──→ 家庭内万器
 :8900      ├─ Mina API ──────→ 音箱大脑
            ├─ eWeLink ───────→ Sonoff 开关（待注册）
            └─ HA REST ───────→ 所有已接入HA的设备（待回家）
```

Gateway 以 FastAPI 构之，聚合多后端，对外暴露统一 REST API。
调用者不必知后端为何——只需 `POST /proxy/voice` 发一指令，
Gateway 自择最佳在线音箱代理执行。

## 六、去枢纽化

> HA 者，枢纽也。万器接入，统一暴露 REST API。
> 然 HA 之耗甚矣：须运行服务器于家中，须配置各平台集成，须保持在线。
> 对于**仅需远程控制小米设备**的用户，HA 是不必要的负担。

遂改 Gateway 架构为**三级凭据链**：

```
L1: config.json 直填 token → 最快，零依赖
L2: .xiaomi_token_cache.json → 自管理缓存，HA只需首次提取
L3: HA .storage/ → 可选回退，有HA则用之
```

**L3 只需运行一次**。首次从 HA 提取 token 后自动缓存至 L2。
此后 Gateway 独立运行，**不再需要 HA 在线**。

此即**渐进式去耦**：
- 有 HA → 自动提取凭据，享受全部功能
- 无 HA → 填入 token 或用 Playwright 登录获取
- 介于二者 → 首次用 HA 提取，之后独立运行

## 七、省思

> 始以为须攻克九大平台，方能远操万器。
> 终乃悟：**一台在线音箱，胜过十个平台API**。

盖音箱者，已在家庭网络之内，已与万器连通，已知万器之名。
余只需对其说一句话，它便替余做一切事。

此即**代理模式**之精妙：
- 不必逐一打通每台设备的云端API
- 不必担心设备是否在线（音箱会在局域网内唤醒）
- 不必处理不同厂商的不同协议

**一言以蔽之：御一箱而制万器，居千里而如咫尺。**
