> **道生一，一生二，二生三，三生万物。**


## 三、多Agent协作协议

### 3.1 认领规则
- 一个Agent **只认领一个道域**
- 认领方式：在该域 `_INDEX.md` 顶部写入 `> 🔒 当前Agent: [对话ID] | 开始时间: [时间]`
- 完成后删除锁标记，写入完成记录

### 3.2 隔离边界
| 规则 | 说明 |
|------|------|
| 文件隔离 | 只修改自己域内文件，跨域需在_INDEX.md中声明依赖 |
| 构建串行 | 同一时刻只有1个Agent执行Gradle构建(共享daemon) |
| 设备独占 | 同一Android设备同一时刻只有1个Agent操作ADB |
| 台式机写入 | 通过SMB(V:/W:/U:)写入台式机时需确认不冲突 |

### 3.3 通信方式
- **协调文件**: `V:\道\AGENTS.md` 本文件 — 全局状态
- **域内状态**: 各域 `_INDEX.md` — 域内进度
- **本机枢纽**: `E:\道\道生一\一生二\双电脑互联\道之统一.md` — 全景映射

## 五、Agent开发模板

每个Agent进入一个道域后，按此流程：

```
1. 读 _INDEX.md → 理解该域资产+现状+目标
2. 写锁 → 标记正在开发
3. 侦察 → 读取该域所有相关文件
4. 去杂 → 删除冗余/过时内容
5. 取精 → 整理核心资产，建立索引
6. 充实 → 根据域的定义，补充缺失内容
7. 验证 → 确保所有链接/引用正确
8. 解锁 → 删除锁标记，写入完成记录
9. 汇报 → 更新道之统一.md中该域的状态
```

## 六、对话结束选项模板

Agent完成一个域的开发后，提供以下选项：
1. **看效果** — 打开该域的_INDEX.md，展示整理结果
2. **下一域** — 认领下一个道域继续开发
3. **去杂** — 执行该域的冗余删除清单
4. **收工** — git commit，记录本次变更
