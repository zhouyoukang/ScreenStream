## 三种连接方式（按优先级）

### 1. WinRM — 最强（推荐）

```powershell
$cred = New-Object PSCredential('administrator',(ConvertTo-SecureString 'wsy057066wsy' -AsPlainText -Force))
Invoke-Command -ComputerName 192.168.31.141 -Credential $cred -ScriptBlock {
    # 在台式机上执行任意PowerShell
    hostname; Get-Process | Select-Object -First 5
}
```

**能力**: 完整PowerShell远程会话，可执行任何命令、管理服务、读写文件、安装软件。

### 2. SSH (plink) — 最稳

```powershell
& "C:\Program Files\PuTTY\plink.exe" -ssh administrator@192.168.31.141 -pw wsy057066wsy -batch "hostname && dir C:\"
```

**能力**: 远程cmd命令执行。`-batch`模式无交互，适合自动化。

### 3. RDP — 看桌面

```powershell
mstsc /v:192.168.31.141
# 凭据已存储(cmdkey)，直接连接
```

**能力**: 完整桌面+音频，60fps。

## 已知限制

| 限制 | 原因 | 绕过方案 |
|------|------|----------|
| Admin shares (C$) 不可用 | RejectUnencryptedAccess=True | 用 FullC/D/E/F 普通共享替代 |
| SSH密钥认证失败 | sshd配置待调试 | 用 plink -pw 密码认证 |
| remote_agent未自启 | 需配置计划任务 | SSH远程启动 |
| WMI/RPC不稳定 | 防火墙配置 | 用WinRM完全替代 |
