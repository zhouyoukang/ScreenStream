# 三界隔离 — Agent指令

## 目录定位
三界分治的运维工具链。天界(用户)、地界(Agent)、人界(共享数据)的隔离与连接。

## 操作此目录时
1. 所有 .ps1 脚本必须用 `pwsh` 运行（非 powershell.exe，避免UTF-8编码问题）
2. 修改脚本后先在天界终端验证：`.\status.ps1`
3. `地界.rdp` 的 `audiomode:i:2` 禁止修改（五感不扰核心配置）
4. 不在脚本中硬编码 windsurf-test 密码

## 关键约束
- **Gradle daemon** 是系统级共享资源，同项目同时只能一个会话构建
- **ADB 设备** 是独占资源，同时只能一个会话操作
- **Git 仓库** 是共享的，两个会话的修改互相可见

## 文件说明
| 文件 | 用途 | 运行位置 |
|------|------|----------|
| enter.ps1 / enter.cmd | 进入地界 | 天界 |
| return.ps1 | 返回天界 | 地界 |
| status.ps1 / status.cmd | 三界状态面板 | 任意 |
| init-agent.ps1 | 地界环境初始化(仅首次) | 地界 |
| 地界.rdp | RDP连接配置 | 天界(双击) |

## 对话结束选项
- 进地界看看 — 打开RDP连到windsurf-test
- 测试隔离 — 验证天界操作不受地界影响
- 完善工具 — 添加更多自动化脚本
- 收工提交 — git commit三界隔离工具链
