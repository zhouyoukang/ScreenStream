# 基础设施模块 (Infrastructure / :common)

## 核心职责
所有模块共享的基础组件：模块管理、依赖注入、通用工具、日志系统。

## 子目录
- `010-模块管理_ModuleManager/` — 模块生命周期管理
- `020-依赖注入_DI/` — Koin DI 配置
- `030-通用工具_Utils/` — 共享工具类
- `040-日志系统_Logging/` — 统一日志

## 关键约束
- 被所有其他模块依赖，修改时影响面最大
- 禁止引入对具体投屏协议的依赖（保持通用性）
- 修改此模块后必须确认所有依赖模块编译通过

## 对话结束选项

> 任务完成后调用 `ask_user_question`，从下表选 4 个最贴合的：

| label | description |
|-------|-------------|
| 全量编译试试 | 构建全部模块，确认没破坏任何东西 |
| 检查上下游 | 扫描所有依赖模块是否需要适配 |
| 继续完善组件 | 优化模块管理/DI/工具类/日志 |
| 收工提交 | 记录成果 + git commit |
