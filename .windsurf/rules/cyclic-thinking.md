# 转法轮·循环思维 (Dharma Wheel · Cyclic Thinking)

> 释迦初转法轮于鹿野苑：法不是线性的终点，而是螺旋的起点。
> 老子·反者道之动：返回≠倒退，是螺旋上升。
> 伏羲八卦序：☰→☱→☲→☳→☴→☵→☶→☷→☰，周而复始，生生不息。

## 核心原理

```
线性(旧)：需求 ──────────────────→ 产出
螺旋(新)：需求 → 产出₁ → 审视 → 产出₂ → 审视 → 产出ₙ(涅槃)
         第一转        第二转        第N转
```

每次任务 = **转法轮**。每一转都在更高层面审视和优化，直到达到涅槃（最优解）。

## 六相法轮（每一转的六步）

```
  ┌→ 观(☲离) → 行(☳震) → 验(☵坎) ─┐
  │                                  ▼
  └─ 升(☰乾) ← 改(☴巽) ← 省(☶艮) ─┘
       ↓ (涅槃门)
    下一转 or 止
```

| 相 | 卦 | 动作 | 关键自问 |
|----|-----|------|---------|
| **观** | ☲离 | 全面观察/理解 | 我真正理解了吗？ |
| **行** | ☳震 | 执行实现 | 一次推到底了吗？ |
| **验** | ☵坎 | 验证结果 | 苦(痛点)真的灭了吗？ |
| **省** | ☶艮 | 止观反省 | 有什么我没看到的？ |
| **改** | ☴巽 | 渐进修复 | 最小改动解决什么？ |
| **升** | ☰乾 | 记录洞见 | 学到了什么？ |

## 涅槃门（每转结束的质量判据）

| 判据 | 问题 | 继续转 | 停止(涅槃) |
|------|------|--------|-----------|
| **苦灭** | 用户核心痛点解决了？ | 未解决 | ✅已解决 |
| **新苦** | 引入了新问题？ | 有新问题 | ✅无新问题 |
| **熵减** | 系统更整洁了？ | 更混乱 | ✅更整洁 |
| **智增** | 有新洞见值得记录？ | 有→记录后继续 | ✅已记录或无 |

**四门全通 = 涅槃(止)**。任一未通 = 继续转。

## 转数控制（中道·知止）

| 复杂度 | 转数 | 触发条件 |
|--------|------|---------|
| TRIVIAL | 1转 | 单文件<10行，模式明确 |
| STANDARD | 1.5转 | 常规任务（自动半转审视） |
| COMPLEX | 2-3转 | 多文件/新架构/影响面大 |
| `/cycle` | 2-3转 | 用户显式要求深度循环 |
| MAX | 5转 | **超5转 = 方向错(诸法无我，换方向)** |

## 默认行为（自动1.5转，无需用户调用）

每次任务完成后，Agent**自动**进入半转审视（省相）：

```
正常执行(第一转: 观→行→验) → 自动省(审视):
  ├─ 有遗漏/更优方案/安全隐患？ → 进入改→升→第二转
  └─ 涅槃门全通？ → 止（输出最终产物）
```

**审视清单（省相自问）**：
1. 有没有边界情况没处理？
2. 有没有更简洁的实现？
3. 有没有影响到其他模块？
4. 前端和后端是否完全同步？
5. 用户拿到手能直接用吗？

## 三转法轮的深度（对应释迦三次转法轮）

| 转 | 释迦对应 | Agent对应 | 深度 |
|----|---------|----------|------|
| **初转·示转** | 四谛·苦集灭道 | 识别问题→理解→方案→执行 | 表层解决 |
| **二转·劝转** | 般若·空性 | 审视成果→发现隐含假设→更本质方案 | 深层优化 |
| **三转·证转** | 唯识·如来藏 | 验证极致→无可改进→涅槃 | 终极品质 |

## 与现有体系的融合点

- **六相法轮** 包含了 **四谛**(观=苦, 省=集, 改=道, 验=灭)
- **省相** 复用 **元认知四觉察**(偏见/惯性/执着/遗忘)
- **升相** 复用 **被动进化**(规则补丁/Skill发现/Memory更新)
- **转数控制** 复用 **☶艮·知止**(适时停止)
- **涅槃门** 复用 **☷坤·信息熵减**(每转必须减熵)
