# 公网部署守卫 (Public Deploy Guard)

> **核心原则**: 阿里云/aiotvr.xyz 是已有基础设施的**参考信息**，不是每次任务的**执行目标**。

## 绝对禁止（无例外）

1. **禁止自动部署到阿里云** — 除非用户**明确说**"部署到阿里云/公网/aiotvr.xyz"
2. **禁止自动检查aiotvr.xyz健康** — 除非用户**明确要求**检查公网状态
3. **禁止自动修复FRP隧道** — 除非用户**明确要求**修复公网连接
4. **禁止自动SSH到阿里云服务器** — 除非用户**明确要求**服务器操作
5. **禁止把"公网服务端口"当作本地开发任务的一部分** — 公网端口表仅供参考

## 判断规则

| 用户说 | Agent做 |
|--------|---------|
| "部署到阿里云/公网" | ✅ 执行部署 |
| "检查公网状态" | ✅ 检查aiotvr.xyz |
| "修复FRP隧道" | ✅ 诊断+修复 |
| "开发XX功能" | ❌ 不涉及公网，只做本地开发 |
| "构建APK" | ❌ 只编译+本地部署到手机 |
| "修复Bug" | ❌ 只修本地代码 |
| 被打断/中断自愈 | ❌ 只诊断本地问题(ADB/编译)，不检查公网 |
