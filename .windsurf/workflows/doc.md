# /doc — 文档工作流

## Phase 1: 确定文档类型

| 类型 | 适用场景 | 输出 |
|------|---------|------|
| README | 项目概览 | `README.md` |
| API 文档 | HTTP/函数接口 | `文档/API.md` |
| 架构文档 | 系统设计 | `文档/` |
| ADR | 架构决策记录 | `文档/adr/` |
| 状态文档 | 功能完成状态 | `文档/STATUS.md` |

## Phase 2: 收集信息

// turbo
1. 读取项目结构（`list_dir`）
2. 读取核心代码文件
3. 读取现有文档（`文档/`）
4. 读取 `AGENTS.md`

## Phase 3: 编写文档

**原则**：
- 面向读者：假设读者没有任何上下文
- 示例驱动：每个功能都有可运行的示例
- 保持更新：代码改了文档必须同步
- 简洁：不写读者能从代码中直接看出的内容

**ADR 模板**：
```markdown
# ADR-YYYYMMDD: 标题
## 状态
[提议/接受/废弃]
## 背景
## 决策
## 理由
## 后果
```

## Phase 4: 验证

- 代码示例是否可运行
- 是否覆盖了所有公共接口
- 与现有文档是否有冲突/重复

## Phase 5: 总结

```
## 文档更新完成
- **类型**: [README/API/架构/ADR/状态]
- **文件**: [路径]
- **覆盖范围**: [描述]
```
