# /refactor — 代码重构工作流

## Phase 1: 分析现状（只读）

1. 用 `grep_search` + `read_file` 定位目标代码
2. 识别重构类型：
   - **提取函数** — 长函数（>30行）拆分
   - **消除重复** — DRY 原则
   - **简化条件** — guard clauses + early return
   - **重命名** — 提升语义化
   - **移动方法** — 放到更合适的模块
   - **提取接口** — 抽象共同行为

## Phase 2: 安全评估

1. 确认修改范围：哪些文件、哪些调用点
2. 检查是否有测试覆盖（没有则先写测试）
3. 确认重构不改变外部行为

## Phase 3: 执行重构

**原则**：
- 每次只做**一种**重构手法
- 用 `multi_edit` 一次性完成同一文件的修改
- 同步更新所有调用点
- 保持现有代码风格

**Guard Clauses 模式**（社区最佳实践）：
```
// ❌ 深层嵌套
fun process(input: String?) {
    if (input != null) {
        if (input.isNotEmpty()) {
            // 实际逻辑
        }
    }
}

// ✅ Early Return
fun process(input: String?) {
    if (input.isNullOrEmpty()) return
    // 实际逻辑
}
```

## Phase 4: 验证

1. 检查所有 import 是否更新
2. 检查所有调用点是否同步
3. 如果有构建系统 → 编译验证
4. 如果有测试 → 运行测试

## Phase 5: 总结

```
## 重构完成
- **手法**: [提取函数/消除重复/...]
- **修改文件**: N 个
- **影响范围**: [描述]
- **验证状态**: 编译通过 / 测试通过 / 待验证
```
