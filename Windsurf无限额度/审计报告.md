## 三、本次修复（v4.0 — 公网VM兼容性专项，8项）

| # | 严重度 | 问题 | 根因 | 修复 |
|---|--------|------|------|------|
| **F1** | 🔴致命 | `ConvertFrom-Json -AsHashtable` PS5.1崩溃 | `-AsHashtable`是PowerShell 7+参数，所有标准Windows VM都是PS5.1 | 改用`PSObject.Properties`手动转换hashtable |
| **F2** | 🔴严重 | hosts重复检测逻辑错误 | `-notmatch`在数组上返回不匹配行(总truthy)，每次运行添加重复条目 | 改为`-not ($content -match ...)`正确的布尔判断 |
| **F3** | 🟡中 | schtasks中netsh无完整路径 | SYSTEM上下文PATH可能不同，计划任务找不到netsh | 改用`$env:SystemRoot\System32\netsh.exe`完整路径 |
| **F4** | 🟡中 | `Resolve-DnsName`不兼容 | Server Core/旧版VM无DnsClient模块 | 改用`[System.Net.Dns]::GetHostAddresses()`（.NET原生,全平台） |
| **F5** | 🔴严重 | 缺少出站18443端口预检 | 云VM防火墙常阻断非标准端口,用户无法诊断失败原因 | 新增Step 1出站TCP预检+详细错误诊断指引 |
| **F6** | 🟡中 | Windsurf搜索路径不全 | 缺少`$env:ProgramFiles`等常见安装路径 | 扩展为6个搜索路径(含ProgramFiles/ProgramFiles(x86)) |
| **F7** | 🔴致命 | server.js windsurf-setup同样PS5.1崩溃 | Web UI一键配置路径也用了`-AsHashtable` | 同F1修复，改用`PSObject.Properties`转换 |
| **F8** | 🟢低 | 连通性结果判断逻辑同F2 | `$testResults -notmatch`在数组上同样总truthy | 改为`-not ($testResults -match ...)`  |

## 五、道·法·术 — 伏羲八卦×道德经×释迦 总评

### 道（为什么） — 老子·道德经
> 天下万物生于有，有生于无。道生一，一生二，二生三，三生万物。

一个Pro账号($15/月)为**一**，通过MITM代理生出**二**(本机+远程VM)，再通过FRP隧道+Web UI+deploy脚本生出**三**(三种入口)，三种入口**生万物**——任意朋友一行命令即可享用Claude/GPT/Gemini全模型。

本质是**☰乾·认知卸载**的极致实践——用户无需理解MITM/gRPC/TLS/DNS任何概念，一行命令，道法自然。

### 法（什么标准） — 释迦·中道

> 苦集灭道。苦=付费壁垒。集=认证依赖第三方。灭=MITM绕过。道=五层拦截中道之法。

| 佛学概念 | 映射 |
|----------|------|
| 诸行无常 | 后端可能宕机/被封号，需接受不确定性 |
| 诸法无我 | 不执着于自建后端(成本>收益)，善用已有资源 |
| 涅槃寂静 | 最终目标=用户无感——看不见代理的存在，只感受到全模型畅通 |
| 中道 | 不走极端(完全免费or完全付费)，卡密共享=成本最小化 |
| 八正道·正业 | 五层拦截每层解决一个具体问题，不多不少 |

### 术（怎么做） — 庄子·庖丁解牛

五层拦截如庖丁之刀，顺应系统骨架(DNS→Chromium→TLS→IDE→gRPC)，游刃有余：

| 层 | 骨节(系统机制) | 刀法(拦截手段) |
|----|---------------|---------------|
| 1 | 系统DNS解析 | hosts劫持→127.0.0.1 |
| 2 | Chromium DNS缓存 | --host-resolver-rules双保险 |
| 3 | TLS证书验证 | LocalMachine\Root自签证书 |
| 4 | IDE配置系统 | settings.json proxyStrictSSL=false |
| 5 | gRPC通信 | MITM代理→后端→auth_token |

> 良庖岁更刀，割也；族庖月更刀，折也。今臣之刀十九年矣——此证书有效期至2036年，十年不更刀。
