# Windsurf 无限额度 — 依赖链深度审计

> 2026-03-06 五感审计：发现所有强限制，不回避任何真相

## 核心真相（一句话）

**当前"无限额度"100%依赖CFW（CodeFreeWindsurf）持续运行并连接其香港后端。用户不需要购买Pro账号，但依赖CFW提供商的Pro账号。这是一个"租用他人Pro"的方案，不是真正的免费。**

---

## 完整依赖链

```
你的Windsurf
    ↓ ① hosts劫持到127.0.0.1
CFW v2.0.5 (PID 34740, 127.0.0.1:443)
    ↓ ② 9条持久TCP连接
CFW后端 (156.225.28.34:5001, 香港)
    ↓ ③ 后端持有 Pro@Windsurf.Pro 账号
    ↓ ④ 返回有效 auth_token (2h TTL)
Language Server 拿到 auth_token
    ↓ ⑤ 直连 (不经代理)
inference.codeium.com → 验证auth_token → 执行推理
```

**断开任何一环 = 推理失败**

## 六大强限制

### ❶ CFW必须持续运行
| 维度 | 状态 |
|------|------|
| 当前 | CFW v2.0.5, PID 34740, 20MB内存 |
| 如果CFW崩溃 | auth_token无法获取/刷新 → 推理API拒绝 |
| Guardian能救吗 | ❌ Guardian只能启动自建代理，但自建代理**不持有Pro凭据** |
| 自建代理能替代吗 | ❌ 自建代理只做透明转发，没有Pro账号的auth_token |

**真相**: `windsurf_proxy.py` 的注释说"不依赖第三方付费授权"是**错误的**。它只是一个透明转发代理，没有Pro账号就只有Free tier的能力。

### ❷ CFW后端必须存活
| 维度 | 状态 |
|------|------|
| 当前后端 | 156.225.28.34:5001 (香港, Vapeline Technology) |
| 历史后端 | 38.175.203.46 (日本东京), 47.108.185.65 (成都阿里云) |
| 如果后端关闭 | CFW无法获取auth_token → 完全不可用 |
| 可控性 | ❌ 完全不可控，由CFW提供商运营 |

### ❸ CFW License Key必须有效
| 维度 | 状态 |
|------|------|
| 当前Key | GG-9ULP-AW3V-3A5N-48UK |
| 如果Key过期/吊销 | CFW后端拒绝服务 → 不可用 |

| 可控性 | ❌ Key的有效性由CFW提供商决定 |

### ❹ 客户端补丁只是"化妆品"
| 补丁 | 实际效果 | 服务端是否认可 |
|------|---------|:-------------:|
| planName:"Pro Ultimate" | UI显示Pro | ❌ 服务端独立验证 |
| isEnterprise=true | 解锁企业UI | ❌ 功能由服务端控制 |
| impersonateTier="ENTERPRISE_SAAS" | metadata注入 | ❌ 服务端不认metadata伪装 |
| U5e永远true | 客户端不限额 | ❌ 服务端独立限额 |

| Premium模型flags | UI显示可用 | ❌ 实际访问靠auth_token |

**真相**: 15个补丁全部是**客户端UI层面**的修改。真正让Premium模型可用的是CFW后端提供的有效auth_token，不是这些补丁。
