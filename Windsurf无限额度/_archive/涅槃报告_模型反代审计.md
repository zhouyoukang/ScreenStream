## 核心结论（先说结果）

### ✅ CFW 不能替换模型 — 架构上不可能

**铁证链:**

| # | 证据 | 含义 |
|---|------|------|
| E1 | `hosts`仅劫持 `server.self-serve.windsurf.com` + `server.codeium.com` | 只有auth API被拦截 |
| E2 | `inference.codeium.com` **不在hosts中** | 推理API直连Codeium |
| E3 | CFW(PID 9592) → 38.175.203.46:5001 (13条TCP) | CFW只连后端，不连推理服务器 |
| E4 | Language Server(PID 40148) → 35.223.238.178:443 (直连) | 推理请求绕过CFW |
| E5 | CFW内存 `target_host: "server.self-serve.windsurf.com"` | 确认只代理auth域名 |

```
实际数据流:
  Auth:      LanguageServer → CFW(:443) → Backend(38.175.203.46) → 获取auth_token
  Inference:  LanguageServer → inference.codeium.com(35.223.238.178) 直连 ← 模型选择在这里
                                    ↑
                            CFW完全不经过这条路径
```

**CFW后端永远看不到推理请求中的模型参数，物理上不可能替换模型。**

### ⚠️ 但SWE-1.5本身是路由模型

用户当前使用 **SWE-1.5 Fast** (0积分) — 这是Windsurf官方的元模型/路由器：
- SWE-1.5 **不是直接调用某个固定模型**，而是由Codeium服务端根据任务动态路由
- 路由算法可能将简单任务分配给便宜模型，复杂任务分配给强模型
- 这是 **Windsurf/Codeium的设计行为**，与CFW无关

### SWE-1.5 路由本质

SWE-1.5 是 **Windsurf自研的模型路由器**(类似OpenAI的GPT-4o路由):
- 服务端根据任务类型、复杂度、上下文长度动态选择底层模型
- 免费(0积分)因为使用成本优化路由 — **可能用较便宜的模型处理简单任务**
- 这是Codeium官方设计，所有用户(包括正版Pro)都一样
- **与CFW无关**

## ☶艮·省 — 发现的问题

### 🔴 P1: 用户使用免费路由模型，未享受无限积分优势
- **现状**: 使用SWE-1.5 Fast (0积分)，底层模型由服务端路由
- **问题**: 有无限积分却用免费路由，相当于开法拉利走乡间小路
- **修复**: 切换到 Claude Sonnet 4.5 (3x积分) 或更强模型，反正积分无限

### 🟡 P2: CFW截图显示请求/会话 8,633/323
- **观察**: 截图中 8633/323 vs 内存提取 12842 — 说明截图是早期状态
- **影响**: 无，纯信息性

### 🟡 P3: 阿里云中枢不可达
- **现象**: `report_state` 超时 (SSL handshake timeout to aiotvr.xyz)
- **影响**: CFW状态上报失败，不影响实际使用
- **原因**: 阿里云服务器sshd/nginx挂起(已知问题)

### 🟢 P4: CFW连接到日本节点(38.175.203.46)
- **观察**: v2.0.6有6个后端节点，当前连到日本
- **影响**: auth延迟略高，但不影响推理延迟(推理直连Codeium美国)

### 🟢 P5: 版本号不一致
- **文件名**: CodeFreeWindsurf-x64-2.0.5.exe
- **内部版本**: v2.0.6 (ProductVersion)
- **影响**: 无，正常增量更新
