## 四、老子道德经 · 根因分析

### 四谛分析

| 谛 | 内容 |
|----|------|
| **苦** | CFW授权码绑定单台电脑的Device Code(dc字段)，多机无法共享 |
| **集** | CFW v2.0.3 Client JWT包含`dc`(设备码)字段，后端校验`key+dc`唯一绑定 |
| **灭** | 所有电脑通过阿里云中枢→FRP隧道→同一台台式机的CFW，共享同一个dc |
| **道** | 阿里云(aiotvr.xyz:18443) → FRP → 台式机(127.0.0.1:443/CFW) → CFW后端 |

### 道·法·术 三层解构

| 层 | 哲学 | 本项目对应 |
|----|------|-----------|
| **道** | 道法自然 — 不对抗系统，顺势而为 | 不破解CFW协议，而是共享CFW实例（如水之利万物而不争） |
| **法** | 中道 — 在约束中找最优解 | 五层拦截 = DNS→Chromium→TLS→IDE→gRPC，每层最小改动 |
| **术** | 庖丁解牛 — 顺应系统骨架 | hosts劫持+证书信任+portproxy+settings.json+客户端patch |

### 上善若水 · 架构哲学

```
水善利万物而不争，处众人之所恶，故几于道。
                                    — 《道德经》第八章

本方案如水:
  不争 — 不与CFW协议对抗，顺势共享
  处下 — 在系统最底层(DNS/hosts)做最轻量拦截
  利物 — 阿里云中枢利万机而不争
  善变 — Guardian守护如水适形，CFW崩则切自建代理
```

### 反者道之动 · 辩证真相

> 曲则全，枉则直，洼则盈，敝则新。

**辩证一**: 15个客户端补丁 = "化妆品"（UI层面），真正的"无限额度"来自CFW后端的Pro账号auth_token。但"化妆品"有其价值——隐藏Free tier限制提示，让用户体验无感。

**辩证二**: 自建代理(`windsurf_proxy.py`)不能替代CFW，但它是CFW不可用时的降级兜底，且其gRPC嗅探能力为认知储备提供了工具。

**辩证三**: 项目名"无限额度"是结果描述而非承诺——前提是CFW后端持续可用。`DEPENDENCY_AUDIT.md`已诚实记录这一真相。

## 五、完整架构（当前运行拓扑）

```
┌─ 任意新电脑 ──────────────────────────────────────────┐
│  deploy_vm.ps1 一键部署:                                │
│    hosts → 127.0.0.1 server.*.windsurf.com             │
│    portproxy → aiotvr.xyz:18443                        │
│    证书 → LocalMachine\Root                            │
│    patch → 15项客户端补丁                               │
│    guardian → 计划任务自愈                               │
└──────────┬────────────────────────────────────────────┘
           │ TLS (portproxy)
           ▼
┌─ 阿里云 aiotvr.xyz ──────────────────────────────────┐
│  auth_hub.py :18800 (授权中枢)                         │
│    /hub/api/health → 健康检查                          │
│    /hub/api/register → 客户端注册                      │
│    /agent/ → 部署脚本+证书分发                         │
│  Nginx 反代 :443 → /hub/ /agent/                      │
│  FRP :18443 → 台式机:443                              │
└──────────┬────────────────────────────────────────────┘
           │ FRP隧道
           ▼
┌─ 台式机 (192.168.31.141) ────────────────────────────┐
│  CFW v2.0.5 :443 (PID 41892)                          │
│    → 156.225.28.34:5001 (香港后端)                     │
│    → Pro@Windsurf.Pro 账号                             │
│    → auth_token (2h TTL, 自动刷新)                     │
└──────────┬────────────────────────────────────────────┘
           │ 直连 (不经代理)
           ▼
┌─ Codeium ────────────────────────────────────────────┐
│  inference.codeium.com                                │
│    验证auth_token → 执行LLM推理                       │
│    Claude / GPT / Gemini 等模型                       │
└──────────────────────────────────────────────────────┘
```

### 多机共享原理

```
笔记本 ──portproxy──→ 192.168.31.141:443 (台式机CFW) ← LAN直连
台式机 ────────────→ 127.0.0.1:443 (本地CFW)         ← 本地
新VM   ──portproxy──→ aiotvr.xyz:18443 (FRP隧道)     ← 公网
```

**关键**: 所有电脑的Windsurf请求最终都到达同一个CFW实例，使用同一个dc(设备码)，从CFW后端视角看只有一台设备。
