**关键发现**:

- **`.@=7`** — 15MB可执行段，包含全部加密载荷+解壳代码
- **入口点在`.@=7`段** — 解壳stub先执行，解密后跳转到`.text`

### 4.4 asInvoker的重要意义

CFW需要管理员权限来:
1. 修改 `C:\Windows\System32\drivers\etc\hosts`
2. 安装自签TLS证书到 `Root CA`
3. 创建端口映射 `netsh interface portproxy`

windsurf-LG标记为`asInvoker`(不需管理员)，这意味着它可能:
1. **不修改hosts** — 使用其他DNS劫持方式(如Chromium `--host-resolver-rules`)
2. **不安装系统证书** — 使用用户级证书或直接修改Windsurf的SSL设置
3. **不用portproxy** — 直接在用户空间监听端口
4. **或者**: 自动请求UAC提升(manifest说asInvoker但可能内部调ShellExecute)

## 八、最终结论

### 8.1 windsurf-LG 是什么

windsurf-LG 是一个**与CFW(CodeFreeWindsurf)完全独立的Windsurf破解工具**:
- 由不同团队开发(不同语言Go vs Rust，不同Key格式)
- 使用自定义加壳保护(重度加密，entropy≈8.0)
- 以`asInvoker`运行(不需管理员)
- 提供GUI界面
- 同样依赖后端服务器(MITM代理模式)

### 8.2 核心问题

**与CFW本质相同**: 都是"租用他人Pro账号"的方案。
- CFW后端: 香港/成都/上海多节点
- LG后端: 未知(加密，需运行时确认)
- 两者都**依赖第三方后端提供auth_token**，断开即不可用

### 8.3 安全评级

```
CFW v2.0.5 (已逆向验证):  风险 5/10  → 可控(已知架构/后端/协议)
windsurf-LG v1.0.0.11:    风险 7/10  → 不可控(完全加密/匿名/未验证)
```

### 8.4 建议策略

| 优先级 | 策略 | 说明 |
|:------:|------|------|
| 1 | **继续使用CFW** | 已深度逆向，风险已知可控 |
| 2 | **LG仅作备用** | CFW不可用时在沙箱中使用 |
| 3 | **动态分析LG** | 如需长期使用，必须先在VM中抓包验证 |
| 4 | **不要同时运行** | 两套MITM代理会冲突(都监听:443) |
