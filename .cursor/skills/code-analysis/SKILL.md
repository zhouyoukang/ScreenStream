# 代码分析 Skill

> **☰乾·认知卸载** — 帮助用户理解代码结构，承担认知负担

## 概述

- **用途**: 深度分析代码结构、依赖关系、架构模式
- **触发**: 用户要求分析、理解、解释代码时
- **卦象**: ☰乾（认知卸载）

## 分析维度

| 维度 | 内容 |
|------|------|
| 结构 | 文件组织、模块划分 |
| 依赖 | import/require、API调用链 |
| 模式 | 设计模式、架构风格 |
| 质量 | 代码复杂度、潜在问题 |

## 输出格式

```markdown
## 结构概览
[文件树/模块关系]
## 核心发现
[关键发现1, 2, 3]
## 建议
[改进建议]
```

## 使用示例

用户: "分析这个模块的代码结构"
→ 读取关键文件 → 结构化分析 → 输出报告

---

## 前置条件

- [ ] 读取相关源代码文件
- [ ] 理解项目上下文
- [ ] 识别关键依赖

## 步骤

1. **收集**: 读取目标文件，理解代码结构
2. **分析**: 识别模块、依赖、模式
3. **诊断**: 发现潜在问题
4. **报告**: 结构化输出分析结果

## 注意事项

- 复杂代码先给概要，再按需展开细节
- 发现问题要明确指出位置和修复建议
- 输出要精简，避免冗余
