# 代码重构 Skill

> **☴巽·渐进渗透** — 最小变更、单变量修复、增量进化

## 概述

- **用途**: 安全重构代码，改善质量而不引入回归
- **触发**: 用户要求重构、改进代码时
- **卦象**: ☴巽（渐进渗透）

## 重构原则

1. **最小变更**: 每次只改一件事
2. **增量进化**: 小步前进，可随时回滚
3. **可验证**: 每次变更后验证功能正常

## 安全检查清单

- [ ] 理解原代码意图
- [ ] 编写/运行测试用例
- [ ] 变更前后行为一致
- [ ] 不引入新依赖
- [ ] 保持向后兼容

## 重构类型

| 类型 | 描述 |
|------|------|
| 命名优化 | 变量/函数/类名更清晰 |
| 函数提取 | 长函数拆分为小函数 |
| 消除重复 | 提取公共逻辑 |
| 简化条件 | 复杂条件简化为易读形式 |
| 迁移语法 | 旧语法迁移到新语法 |

## 注意事项

- 禁止大跃进式重构
- 稳定代码不轻易触碰
- 有测试覆盖才敢大改
- 重构前备份，变更可追溯
