# 道之隔离架构 — 三界分治，五感不扰

## 无感模式（默认）

**用户永远在天界（zhouyoukang），隔离在后台自动发生。**

- 12/12 开发工具在主账号中完美可用
- Git 安全网自动保护：Agent 操作前自动快照，出错可回滚
- Windsurf Worktree 提供分支级隔离
- windsurf-test 账号为备用恢复通道，日常不需切换

```
git-safety-net.ps1 -Action snapshot   # Agent操作前自动调用
git-safety-net.ps1 -Action rollback   # 出错时一键回滚
git-safety-net.ps1 -Action list       # 查看所有快照
```

## 切换模式（备用）

仅当需要完全隔离的环境时使用：`Win+L → windsurf-test → 0066`

## 为什么需要三界隔离？

| 问题 | 天界独占时 | 三界分治后 |
|------|-----------|-----------|
| Gradle构建吃满CPU | 用户卡顿 | 地界构建，天界流畅 |
| Agent误删文件 | 用户数据丢失 | Git回滚，天界无感 |
| 安装包/依赖冲突 | 污染用户环境 | 地界隔离，天界纯净 |
| 终端卡死 | 整个IDE卡死 | 只影响地界，天界不受干扰 |
| 系统崩溃(BSOD) | 用户工作丢失 | 地界可牺牲，天界会话保留 |

## 快速开始

### 首次使用

```powershell
# 1. 进入地界 (打开RDP窗口)
.\enter.ps1

# 2. 在地界中初始化开发环境 (仅需一次)
.\init-agent.ps1

# 3. 查看三界状态
.\status.ps1
```

### 日常使用

```powershell
# 方式A: RDP窗口 (推荐 — 天界地界同时可见)
.\enter.ps1 -Mode rdp

# 方式B: 快速切换 (全屏切换，天界后台保留)
.\enter.ps1 -Mode switch

# 方式C: 双击RDP文件
# 直接双击 地界.rdp
```

### 返回天界

```powershell
# 在地界中运行:
.\return.ps1

# 或: Ctrl+Alt+Del → 切换用户 → zhouyoukang
# 或: 关闭RDP窗口 (地界继续运行)
```

## 文件清单

| 文件 | 用途 |
|------|------|
| `地界.rdp` | 双击即连地界，audiomode=2(静音)防声音干扰 |
| `enter.ps1` | 智能进入地界 (auto/rdp/switch 三模式) |
| `return.ps1` | 从地界切回天界 |
| `status.ps1` | 三界状态面板 (会话/数据/连接/资源) |
| `init-agent.ps1` | 地界首次环境初始化 (Git/Python/ADB/Windsurf) |

## 技术细节

### 隔离保证

- **进程隔离**: Windows会话天然隔离，不同用户的进程互不可见
- **桌面隔离**: 每个会话有独立桌面、窗口站、剪贴板
- **声音隔离**: RDP设置 `audiomode:i:2` 禁止地界声音传到天界
- **键鼠隔离**: RDP窗口内的键鼠事件不影响天界 (`keyboardhook:i:2`)

### 共享保证

- **文件共享**: E:\道\ 和 E:\github\ 的 ACL 允许 Authenticated Users 读写
- **SMB共享**: C/D/E 盘均已配置 SMB 共享
- **Git共享**: 同一个 .git 仓库，两个会话看到相同的代码

### 并发支持

- **RDP Wrapper**: 已安装，支持 console + RDP 并发会话
- **fSingleSessionPerUser=0**: 允许同用户多会话（此处不同用户，天然支持）

### 回滚机制

1. **代码回滚**: `git checkout .` 或 `git reset --hard`
2. **系统还原**: Windows 系统还原点
3. **账号重建**: 删除 windsurf-test → 重建 → 重新初始化

## 注意事项

- 首次 RDP 连接需输入 windsurf-test 密码，勾选"记住凭据"后免密
- 地界中的 Gradle 构建会共享 daemon（同项目同时只能一个在构建）
- ADB 设备是系统级资源，同时只有一个会话能操作同一设备
