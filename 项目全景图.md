## 分离与联系

### 依赖关系图

```
                    ┌──────────────────┐
                    │ A. ScreenStream   │ ← 上游：70+ HTTP API
                    │   (Kotlin/Android)│
                    └──┬───────────┬───┘
                       │           │
            封装API    │           │  路由挂载
                       ▼           ▼
              ┌────────────┐  ┌────────────┐
              │ 手机操控库  │  │ 智能家居    │
              │ (Python)    │  │ (Python)    │
              └────────────┘  └────────────┘
                                    │
                                    │ 微信控制
                                    ▼
                            ┌────────────┐
                            │ 微信公众号  │
                            └────────────┘

┌────────────┐   完全独立，零依赖
│ 远程桌面    │
│ 二手书项目  │
│ C/D/E/F 域  │
└────────────┘
```

### 联系（跨项目同步点）

| 变更源 | 影响目标 | 同步动作 |
|--------|---------|---------|
| `反向控制/` API 变更 | `手机操控库/` | 更新 `phone_lib.py` 对应方法 |
| `智能家居/网关服务/` | 微信公众号功能 | 测试微信消息→设备控制链路 |
| `投屏链路/index.html` | 智能家居面板 | 检查 Alt+Shift+H 面板是否正常 |

### 分离（多Agent隔离）

| 原则 | 说明 |
|------|------|
| **每个目录有AGENTS.md** | 定义该区域的文件权限边界 |
| **外部项目物理隔离** | Junction只是快捷方式，删除不影响原项目 |
| **端口固定不冲突** | SS:8080-8099 · SmartHome:8900 · RemoteDesktop:9903 |
| **构建串行** | 同时只有1个Agent执行Gradle构建 |
| **设备独占** | 同一Android设备同时只有1个Agent操作ADB |
