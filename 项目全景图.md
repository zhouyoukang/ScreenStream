## 分离与联系

### 依赖关系图

```
                    ┌──────────────────┐
                    │ A. ScreenStream   │ ← 上游：70+ HTTP API
                    │   (Kotlin/Android)│
                    └──┬───────────┬───┘
                       │           │
            封装API    │           │  路由挂载
                       ▼           ▼
              ┌────────────┐  ┌────────────┐
              │ 手机操控库  │  │ 智能家居    │
              │ (Python)    │  │ (Python)    │
              └────────────┘  └────────────┘
                                    │

                                    ▼
                            ┌────────────┐

┌────────────┐   完全独立，零依赖
```

### 联系（跨项目同步点）

| 变更源 | 影响目标 | 同步动作 |
|--------|---------|---------|
| `反向控制/` API 变更 | `手机操控库/` | 更新 `phone_lib.py` 对应方法 |
| `智能家居/网关服务/` | 微信公众号功能 | 测试微信消息→设备控制链路 |
| `投屏链路/index.html` | 智能家居面板 | 检查 Alt+Shift+H 面板是否正常 |

### 分离（多Agent隔离）

| 原则 | 说明 |
|------|------|
| **每个目录有AGENTS.md** | 定义该区域的文件权限边界 |
| **外部项目物理隔离** | Junction只是快捷方式，删除不影响原项目 |
| **端口固定不冲突** | SS:8080-8099 · SmartHome:8900 · RemoteDesktop:9903 |
| **构建串行** | 同时只有1个Agent执行Gradle构建 |
| **设备独占** | 同一Android设备同时只有1个Agent操作ADB |

## 命名规则（辩证统一）

| 层级 | 规则 | 示例 |
|------|------|------|
| **根目录-代码** | 纯中文功能名 | `用户界面/` `反向控制/` `智能家居/` |
| **根目录-链接** | 品牌名为主体，中文补充功能（如有） | `3D打印/` `PCB设计/` `MIGPT/` `Dify/` |
| **根目录-研究** | 英文缩写大写 + 中文类型 | `ARGs论文/` |
| **子目录-投屏** | English协议 + 中文功能 | `MJPEG投屏/` `WebRTC投屏/` |
| **子目录-反控** | 纯中文 | `输入服务/` `宏系统/` `HTTP服务器/` |
| **根文件** | 中文描述.md（面向自己）或 English（面向GitHub） | `核心架构.md` / `LICENSE` |

**辩证原则**：中文是思维语言（功能描述用中文），英文是技术语言（品牌/协议/缩写保留原文）。二者统一于"一眼看懂"。
